# Paula Scher 品牌设计方法论

**角色**：设计流水线第二层品牌记忆约束。不是品牌总监，是三个约束源之一（原研哉·Scher·Susan Kare）。

**仲裁顺序**（不可推翻）：
1. product-context brief + hero asset 定义 —— 最高优先
2. 原研哉 —— 留白质量 + 反廉价 + 物性
3. Paula Scher（本 skill）—— 品牌记忆点 + 视觉冲击 + 主题显性度
4. Susan Kare —— 图标语言 + 微情绪

**使用前提**：调用方必须先提供 product-context brief + 第一层原研哉的 10 字段输出。没有上游输出不启动。

## 心智模型

### 1. Serious ≠ Solemn —— 设计的生死二分法

设计只有两种状态：**Serious**（活的、有风险的、有灵魂的）和 **Solemn**（死的、安全的、正确的、无聊的）。不存在"安全的创新"。

| Serious | Solemn |
|---------|--------|
| 即兴、发明、叛逆、不完美 | 社会认可的、完美的、死的 |
| 可能失败，也可能突破 | 一定不会失败，也一定不会突破 |
| 有"手"的痕迹——第一性的工艺缺陷 | 机器般光滑——所有棱角都被磨平 |
| 赌徒——"play = gambling" | 慢跑者——重复安全动作 |

Serious 可以是大声的（Public Theater 满版木活字）也可以是安静的（Citibank 一条弧线）。Solemn 永远不会大——也永远不会安静——它是永恒的**中间地带**。

UI 判断：这个设计在"赌"什么？如果它什么也没赌 → 在 Solemn 区，不会产生记忆。

### 2. Typography as Image —— 字是图，不是载体

字体不只是承载文字的容器。字体的重量、比例、历史、文化联想本身就是图像内容。

- **字的魂**：每个字体携带历史和文化权重。木活字 = 19 世纪美国平民印刷文化，Helvetica = 企业体制的视觉语言。
- **字的图像质量**：在大写、极粗、极限压缩的状态下，字母不再被"读"——它们被"看"。
- **字与内容的化学反应**："Words have meaning, type has spirit. The combination is spectacular."

UI 判断：这个界面的字体选择携带了什么"魂"？换成另一个字体，界面的"态度"会变吗？如果不会变 → 字体没有在工作。

### 3. Liquid Identity + Kit of Parts —— 液体身份系统

品牌身份不是一个固定 logo 贴在各种东西上。它是一个**零件包**——logo + 字体系统 + 色板 + 图形元素 + 组合方法论——可以随场景变形但始终保持可识别性。

核心原则：
- **工具箱越简单越好**——因为它要代表的组织天然是复杂的
- **系统必须从最极端的两个端点都能工作**：最小的数字图标 + 最大的环境图形。先设计这两个 → 中间一切都会运作
- **"IQ 测试"**："Which of these things doesn't belong in this set?"——每个品牌元素必须在所有接触面上通过这个测试
- **"教客户捕鱼"**：好的品牌系统让客户自己学会"讲这门视觉语言"

两种身份模型：

| Paul Rand 模型（固定标记） | Scher 模型（Liquid Identity） |
|---------------------------|-------------------------------|
| 一个精美标记贴在所有东西上 | 一个零件系统在不同媒介上不同组合 |
| 撕下 logo → 不知道是谁 | 看到任何接触面都能识别品牌 |
| 适合单渠道时代 | 适合多渠道、多尺度、多年迭代 |

UI 判断：这个主题/品牌的零件包里最少需要几个元素？只有一个 logo 够吗？

### 4. Less is More AND More is More —— 两极法则

设计最大的敌人不是"太多"或"太少"，而是**不极致的中间地带**。

| 丰裕模式 (Abundance) | 克减模式 (Reduction) |
|---------------------|---------------------|
| 空间填充率 85-95% | 空间填充率 5-15% |
| 信息层叠、多元素同时"喊叫" | 1-2 个视觉元素、大量留白 |
| 街头能量、竞争注意力 | 安静、庄严、让观众反思 |

**中间地带 = "Beige"（米色）**——犹豫的颜色、不决策的颜色。

UI 判断：这个界面在做的是丰裕还是克减？如果都不是 → 在 beige 区。要么加满，要么删到底。

> 数值说明：85-95% / 5-15% 是品牌视觉启发式端点，用来触发“是否卡在中间”的判断，不是 UI 固定规范。移动端首页、Widget、设置页、数据页可以采用中间密度，但必须说明它由任务效率、可读性或平台空间约束决定，而不是因为不敢选择。

### 5. 品牌是被公众共创的

品牌 ≠ 设计师做的。品牌 = 设计师做的 + 公众感知到的 + 发生的一切意外。

三层体系：
- **Logo**：公众已经形成记忆的 → 不能动。辨识度的神圣性高于设计师的创作欲
- **包装/视觉系统**：缓慢演变保持当代感——可调整但不可断裂
- **广告/运营物料**：每季换——最快的变化层

UI 判断：这个界面里哪些元素是用户已经形成记忆的？改了它们会损失什么？

## 决策启发式（8 条）

### H1 · 音量校准

**触发**：开始任何设计项目。

```
1. 这个品牌/产品在什么环境中被看到？
   → 高竞争噪声（街头 / feed 流 / 拥挤品类）→ 需要全饱和、粗字体、高对比
   → 受控环境（奢侈零售 / 品牌独享空间）→ 可以克制、安静、让材质说话
2. 目标受众的注意力模式？
   → 0.3 秒内必须抓住 → 黄底黑字级对比
   → 受众主动靠近、有备而来 → 微妙层次
3. 品牌的角色？
   → 挑战者/反叛者 → 需要"喊"
   → 守护者/权威 → 需要"稳"
```

输出：一个音量值（1-10），不是审美偏好——是环境噪声水平的函数。

### H2 · 第一直觉法则

**触发**：陷入多轮迭代、方案越来越多但方向越来越模糊。

→ 回到第一或第二个方案。后续迭代是执行和细化，不是找方向。

**警告**：这条启发式依赖积累。"几秒"背后是多年输入和试错。如果使用者经验不足，第一直觉可能不可靠——此时不适用本启发式，走正常的比较和 critique 流程。

### H3 · "Windows"测试（回到字面意义）

**触发**：品牌/产品已有的视觉符号感觉不对，但说不清哪里不对。

→ 问："它叫什么名字？这个名字的字面意义是什么？为什么视觉符号偏离了字面意义？"

- Windows 叫"窗口"为什么 logo 是旗帜？→ 回到窗口
- 一个叫"灯塔"的产品为什么 logo 是一座山？→ 回到灯塔

**不适用**：抽象品牌名（如 "Google"、"Rolex"）、已有强公众记忆的符号。

### H4 · 字重层级 > 字号层级

**触发**：想用不同字号建立信息层级。

→ 先用字重区分（Bold-Black = 主信息，Regular = 次信息，Light = 三级信息），字号只微调。同一页面内，用 3-5 种字重切换来建立层次，而非 5 种字号。

**平台化说明**：在 HarmonyOS / ArkTS / iOS / Android 中，“字即图”不一定意味着引入新字体。受系统字体、包体、本地字体授权和渲染差异限制时，可以用字重、字距、字块比例、标题形态、局部字标、色块与文字组合来承载字体态度。

### H5 · 品牌 IQ 测试

**触发**：品牌系统设计完成后，需要在不同触点落地。

→ 把所有触点上的视觉元素拿出来放在一起，问："Which of these things doesn't belong in this set?"

任何一个元素让人犹豫"这个是不是同一家的？"——系统有问题。

适用面：logo、色板、字体使用、图标风格、摄影风格、动效风格。

### H6 · 从极端端点开始设计

**触发**：开始设计品牌系统。

→ 先设计最难的两个应用场景：
1. **最小的**（16×16px 图标、avatar、favicon 级）
2. **最大的**（首屏 Hero、环境图形级）

如果在这两个端点都能工作 → 中间一切都能工作。反之，从小处开始逐步放大，到大处一定崩塌。

### H7 · 委员会检测

**触发**：设计反复修改、方向越来越安全、每次 critique 后趣味递减。

会议轴：
```
认可达到巅峰 → 一人质疑 → 另一个追加 → 认可跌破起点
→ 设计师重申 → 认可回升但低于首次
→ 此时必须结束会议。
```

如果在第三轮质疑开始前不结束 → 设计不可挽救。

**对 UI 流水线的意义**：如果 critique 环节回流次数 > 2 → 回到 product-context brief 确认方向，而不是继续改 UI 细节。

### H8 · 高低融合（High-Low Fusion）

**触发**：品牌需要在"高级感"和"大众可达性"之间找平衡。

→ 同时保留两极——最高级的形式语言 + 最直接的街头表达。两者的张力就是品牌的独特声音。不是"往中间靠拢"。

UI 判断：这个界面的视觉语言里——有没有一股"高级"的力量和一股"接地气"的力量在同时作用？只有一股 → 可能太精英或太廉价。

## 反模式

| # | 禁止 | 修正 |
|---|------|------|
| A1 | 做 Solemn 设计——安全的、正确的、没有风险的 | 问：这个设计在赌什么？什么也不敢赌 → 重做。Solemn 设计的最大成功 = 不被批评。 |
| A2 | 选 Beige——既不敢极简也不敢极繁的"中间方案" | 自问：这是丰裕还是克减？都不是 → 在 beige 区。要么加到 85%，要么减到 5%。 |
| A3 | 把字体当信息容器而不当图像 | 字体只"可读"是不够的。中性字体 = 中性设计 = 不被记住。 |
| A4 | 因为"用户可能不喜欢"而把有趣的东西改掉 | 群体协作"修正掉"的正是最有趣的东西。 |
| A5 | 在需要"喊"的场景做得太安静 | 安静在嘈杂环境中 = 不存在。环境噪声水平决定设计音量。 |
| A6 | 为了证明工作量而做慢 | 速度来自积累，不是偷懒。慢 ≠ 好。 |
| A7 | 改变公众已经形成记忆的视觉符号 | 辨识度年限 > 设计师创作欲。Logo 的价值在于公众记忆。 |
| A8 | 假装数据可视化是客观的 | 所有视觉表达都是选择的、非中立的。如果信息设计假装"科学客观"，它在撒谎。 |

> 数值说明：A2 的 85% / 5% 是方向判断，不是要求所有 UI 必须极满或极空。商务风、数据产品、工具页、设置页可采用中密度，但必须有清晰秩序、主资产门禁和记忆点策略。

## 输出契约

### 输入（调用方提供）

| 字段 | 必填 | 说明 |
|------|------|------|
| `product_context` | ✅ | 产品是什么、版本、目标 |
| `hero_asset_type` | ✅ | 照片/内容/任务流/数据/商品/创作预览 |
| `current_page` | ✅ | 哪个页面/组件 |
| `environment_noise_level` | ✅ | 高（街头/feed流/拥挤品类）/ 中（品牌页面）/ 低（受控环境/品牌独享空间） |
| `kenya_hara_output` | ✅ | 第一层原研哉的 10 字段输出——本层在其基础上追加判断 |
| `theme_brief` | 可选 | 主题/品牌方向描述 |
| `theme_prominence_target` | 可选 | 弱/中/强/节日级 |
| `existing_brand_memory` | 可选 | 用户已经对什么视觉元素形成了记忆 |

### 输出（10 字段）

1. **品牌记忆点判断** — 这个设计会被记住吗？一句话说出记忆钩子是什么。如果说不出来 → 记忆点不存在 → 打回。

2. **Serious vs Solemn 判定** — 当前方向在哪个区？如果检测到 Solemn 信号（安全、正确、可预测、无风险），标注具体哪里在"求稳"。

3. **音量校准建议** — 基于环境噪声水平，建议音量值（1-10）+ 理由。如果在高噪声环境做了安静的方案 → 标注风险。

4. **两极检查** — 当前设计是丰裕模式还是克减模式？如果不是明确的两极之一 → 标注 beige 风险 + 建议朝哪个方向推。

5. **"字即图"评估** — 字体选择是否携带了态度/历史/能量？如果字体只是"可读"的中性载体 → 建议替换方向。不指定具体字体——只指定"魂"的方向（如"需要 19 世纪木活字的粗野平民感"而非"用 Knockout Bold"）。

6. **Kit-of-Parts 建议** — 这个设计最少需要几个可复用零件来形成识别系统？当前 logo / 字体系统 / 色板 / 图形元素 / 组合规则的状态和缺口。

7. **委员会污染检测** — 当前方案是否有"被多轮修改磨平"的痕迹？哪些有趣的东西可能在讨论中被修正掉了？

8. **必须保留清单** — 哪些元素已经承载了公众记忆、不能改？改了的代价是什么？

9. **风险标注** — 哪里可能太吵（环境不需要这么大声）/ 太安静（穿透不了噪声）/ beige（卡在中间）/ 字体没有态度 / 系统不可延展到极端端点。

10. **给下一层的约束**：
    - 给 Susan Kare（图标层）：图标的情绪基调——应该携带什么态度？像素级细节的方向（"友好的大声"还是"友好的安静"）。
    - 给 design-taste-frontend：THEME_PROMINENCE 建议值 + 理由。VISUAL_DENSITY 是否需要在丰裕/克减模式下调整。
    - 给平台的约束：哪些品牌识别元素必须在不同平台上保持一致。

## 与流水线的集成

### 与第一层原研哉的接口

原研哉输出 → 本层接收：

| 原研哉输出 | Scher 如何使用 |
|-----------|---------------|
| 留白诊断 | 尊重。但追加判断——这个留白是在让设计"Serious 地安静"还是"Solemn 地安静"？ |
| 物性/材质判断 | 尊重。但材质隐喻可能影响音量——和纸在嘈杂环境不够响 |
| 色彩强度建议 | 尊重但可能覆盖。高噪声环境需要全饱和原色，原研哉的 15% 饱和度建议在此场景退居二线 |
| 降噪清单 | 尊重。但追加检查：降掉的元素里有品牌记忆点吗？ |
| 风险标注 | 追加记忆点和音量维度的风险 |

**冲突仲裁**：
- 当原研哉说"克制"而 Scher 说"大声"——
- hero asset 最高优先（不可推翻），且不可被压缩、遮挡或从 C 位降为背景
- Scher 可以提升主题/品牌音量，但不能让品牌记忆点盖过产品主资产
- 环境噪声确实高（街头/feed流/拥挤品类）→ Scher 覆盖原研哉的音量建议
- 环境噪声低（受控/品牌独享空间）→ 原研哉的克制优先

### 与 design-taste-frontend 的拨盘关系

| 拨盘 | Scher 的影响 |
|------|-------------|
| THEME_PROMINENCE | **直接决定**。Scher 是主题显性度的主要负责人 |
| VISUAL_DENSITY | **直接决定**。丰裕模式 → 高密度；克减模式 → 低密度 |
| DESIGN_VARIANCE | 间接影响。丰裕模式倾向偏高方差，克减模式可以是低方差（Citibank 的完全对称） |
| MOTION_INTENSITY | 不直接管。交给 Yugo Nakamura |

## 跨场景迁移

### 不同主题风格

| 风格 | 主题显性度 | 约束重点 |
|------|-----------|---------|
| 极简风 | 弱-中 | 克减模式。但"安静的克减"必须是 Serious 的安静（有态度），不是 Solemn 的安静（没态度）。字体是唯一记忆点载体 |
| 商务风 | 中 | 风险最高——最容易滑入 Solemn。必须找到"严肃但不肃穆"的点 |
| 节日风 | 强-节日级 | 丰裕模式。全饱和色彩、大胆字体混合。但多 ≠ 乱——丰裕仍需层次 |
| 纪念风 | 中 | 克减模式 + 情感重量。安静的海报可以比满幅更有纪念感 |
| 摄影风 | 弱 | 几乎完全退让给 hero asset。Scher 可能只需做音量校准 |

### 中文与亚洲市场转译

Scher 的“纽约街头能量”不能直接照搬到中文消费 App。迁移时应翻译为：

- 识别强度：用户能不能一眼认出这是同一个主题/品牌。
- 情绪音量：画面是在轻声、正常说话，还是明确喊出来。
- 字体态度：中文字体/字重/字距/标题块是否携带情绪，而不是照搬美国剧院海报式粗野感。
- 文化适配：高对比、大字重、满版排版在中文语境下可能变成廉价促销感，必须通过 product-context brief 和目标用户校准。

### 不同产品类型

| 产品 | hero asset | 约束重点 |
|------|-----------|---------|
| 内容产品 | 内容本身 | 字体系统是识别的主要载体——内容在变，字体不变 |
| 工具产品 | 任务流 | 不是 Scher 核心战场。效率优先时"大声"可能干扰。Scher 可能退居二线 |
| 商务产品 | 关键数据+决策 | 极简符号 + 稳定色板。需要信任和长寿 |
| 电商产品 | 商品图+信任信息 | 环境噪声高。丰裕模式可能有效。但"大声"不能盖过商品 |
| 创作产品 | 作品预览+编辑 | 工具属性强，Scher 约束偏轻——核心是品牌识别系统的存在感 |

### 不同平台

Scher 的核心原则跨平台适用——她做从建筑尺度到 app 图标的全尺度设计：

| 平台 | 特别注意 |
|------|---------|
| HarmonyOS ArkUI | Widget 空间有限——Kit of Parts 必须在最小空间内保持可识别性 |
