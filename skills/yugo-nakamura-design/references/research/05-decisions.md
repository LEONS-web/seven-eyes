## 一、MoMA "Design and the Elastic Mind" 网站（2008）

### 决策 1：不修复问题，而是放大问题

- **来源**：一手（MoMA 采访）+ 二手（Eye Magazine 评论）
- **可靠性**：高

**背景**：MoMA 的 "Design and the Elastic Mind" 展览使用了"Thought to Action"、"All Together Now!"、"People and Objects" 等模糊分类来组织展品。这些分类有意避免传统策展逻辑，但作为网站导航信息架构却很糟糕——用户很难通过它们找到具体作品。

**问题**：如何为展览做网站？常规做法是用艺术家、媒介、时间线等传统分类（如 MoMA 同期另一个展览 "Color Chart" 网站那样），但这会背叛展览的核心精神——这些项目正是被选中因为它们挑战了简单分类。

**决策**：中村选择**不"修复"策展的模糊性，而是把它变成网站的核心交互语言**。他用漂浮的图像气泡取代了静态导航，每个气泡触发一组动态连线——"scenario"、"tinkering"、"interfaces"、"collective"、"senses"——引导用户以非线性的、联想式的方式探索作品。

**反决策分析**：为什么不用传统分类法？
1. 如果"修复"了策展的弱点，网站就和展览内容产生矛盾——展览的主题恰恰是"弹性思维"
2. 常规方案会抹杀展览的独特性，让网站沦为普通的博物馆目录
3. 中村主动选择了"把问题放大"——当模糊性本身成为交互机制的一部分时，它从 bug 变成了 feature

**中村自述**："我们首先想到网站需要覆盖的两个功能。一个是网站本身成为一种跟随'弹性思维'主题的创造性表达。另一个是确保它也能作为信息型网站运转。最终结果不是一个融合的整体，而是一个设计，提出了两个并行存在的不同层次。"（MoMA 采访，2010）

**结果**：Eye Magazine 评价"a lesser designer might have tried to 'fix' the issue... But instead, Nakamura chooses -- quite brilliantly -- to make the problem worse."

**可提取规则**：
- **规则 1-A（反常规修复）**：当客户给的约束本身就是内容的核心特质时，不要"修复"它。放大它、把它变成交互语言的一部分。模糊性可以被设计成一个 feature 而非 bug。
- **规则 1-B（双层结构）**：当网站同时需要"表达性"和"信息性"两种功能时，不强行融合成单一体验，而是让两层并行存在——一层是诗意的、联想的，另一层是结构化的、可检索的。

### 决策 2：让网站本身成为展览品质的作品

- **来源**：一手（MoMA 采访）
- **可靠性**：高

**背景**：MoMA 策展人 Paola Antonelli 在选择设计公司时，看了三家公司的约二十个链接。当看到 yugop.com（白页上二维球体像物理模拟般落下）时，她当场拍板选 tha ltd.。中村被选中不是因为他做过博物馆网站，而是因为他的个人实验作品本身就有"展览品质"。

**问题**：展览网站通常被视为"宣传品"而非"展品"。如何让网站不降格为只是展品的容器？

**决策**：中村设定了一个极高的内部标准——"我们真的非常努力地工作，确保我们的网站可以和这些艺术作品一样好，或接近它们。"（MoMA 采访，2010）他不是把网站当作展览的附属品，而是当作展览本身的数字延伸——一件和其他展品处于同等品质水平的创作。

**反决策分析**：为什么不像大多数博物馆网站那样做信息型设计？
- 展览主题是"弹性思维"——一个纯粹信息型的 HTML 网站会违背这个主题
- 如果只是做常规的策展导航，tha ltd. 的独特能力（物理模拟、动态系统、诗意的交互）没有发挥空间
- 中村的判断是：当展览内容本身的品质足够高时，包装它的容器必须匹配这个高度

**中村自述**："博物馆网站和商业网站之间没有特别的不同，除了具体的任务要求。无论如何，我们尝试平衡各元素并给出自己的解决方案。"（MoMA 采访，2010）

**可提取规则**：
- **规则 2**：当内容和容器存在品质落差时，容器必须匹配内容的品质高度。不要满足于做"够用"的信息型设计——如果展览、产品、品牌本身有艺术野心，它的数字呈现也必须达到同一水准。

## 二、KDDI INFOBAR A01 / A02 智能手机 UI（2011-2013）

### 决策 3：用 Flash 可运行原型替代规格文档

- **来源**：一手（ITmedia トークセッション报道，中村现场发言）+ 二手（KDDI 砂原企画責任者自述）
- **可靠性**：高

**背景**：2010 年前后 Android UI 还很稚嫩，图标设计缺乏一致性。KDDI 委托深泽直人（产品设计）和中村勇吾（UI 设计）合作开发 INFOBAR A01——一个需要从零设计全部 UI 的 Android 手机。中村需要在深泽的硬件设计理念和 Android 系统约束之间找到一个 UI 表达。

**问题**：UI 的动态行为（弹性、惯性和交互手感）是感性的、难以用文字传达的。传统流程是：设计方向制造商描述 -> 制造商工程团队实现 -> 来回调整。这个过程对"微妙的动效质感"极为低效。

**决策**：中村在 tha ltd. 用 Flash 制作了完整的可交互 Demo（"iida UI Virtual Touch"），然后把带有注释的 Flash 代码文件直接交给制造商 Sharp。KDDI 的砂原描述："普通软件是按规格书编程的，但 iida UI 有无法用规格书表达的动态和交互，所以先由中村方面用 Flash 制作交互 Demo，然后交给 HTC（A02 制造商），把 Flash 代码替换成 Java，忠实还原 Demo 的动态。"

**中村自述**："'这里请更轻柔一点，再锐利一点'这样说然后看对方交上来的东西，再'请稍微退回去一点'这样做非常花时间。如果能直接交付（代码），就能缩短流程。"（ITmedia, 2011）

**反决策分析**：为什么不走常规的"写文档-交给工程师实现"流程？
1. 文字和静态图无法描述"ぷにぷに（弹弹的质感）"——中村在 A02 中追求的是像果冻一样受压变形的图标，这需要算法而非图形
2. 10 个工程师会对感性描述产生 10 种不同理解——中村选择消除这个解释空间
3. 可运行原型的价值在于：不是"描述行为"，而是"定义行为 = 代码就是规格"

**结果**：INFOBAR A01 获得广泛好评，iida UI 成为差异化的核心卖点。Flash Demo 后来被直接移植为网站上的 "iida UI Virtual Touch" 供用户体验。

**可提取规则**：
- **规则 3-A（可运行原型替代文档）**：当设计涉及动态行为、物理手感、时序节奏时，用可运行原型替代文本文档。原型不是"展示想法"的工具，原型就是规格本身。
- **规则 3-B（代码带注释交付）**：把带有注释的代码直接交给工程实现方，减少"解释-翻译-实现"过程中的信息损失。设计意图编码为算法逻辑。

### 决策 4：从图标到模块与构图（Modules & Composition）

- **来源**：一手（中村在 ITmedia トークセッション的发言）+ 二手（KDDI 砂原说明）
- **可靠性**：高

**背景**：2010-2011 年，Android 的 UI 范式是 iOS 式的图标网格。图标风格不统一是 Android 的普遍问题。INFOBAR 需要一个独特的 UI 语言。

**问题**：智能手机的传统逻辑是"图标 = 应用入口，点开才能看到内容"。这带来两个问题：(1) 用户需要逐一点开才能获取信息，(2) 不同应用的图标风格不一致破坏了视觉统一性。

**决策**：中村提出了一个颠覆性的 UI 范式转变——从"アイコン（图标）"到"モジュール（模块）"再到"コンポジション（构图）"。核心概念：
- **一览性（glanceability）**：界面像报纸版面一样，不需要点开就能看到内容。"普通智能手机有图标，不开应用就不知道内容，但我们想让人'啪'地一下看到。"
- **模块化**：各种功能（音乐播放器、Facebook feed、天气）作为可定制的"面板"平铺在纵向滚动页面上——像音响组件（卡带、均衡器）整齐排列
- **构图 = 自我表达**：用户可以自由布局面板，通过面板的排列和色彩创造个人"表情"

**反决策分析**：为什么不像 Windows Phone 7（同期也采用 tile 设计）那样用固定网格？
1. 中村的意象来源是"オーディオコンポ"——音响组件，每个功能模块有自己的形状和尺寸，但整体协调
2. 固定的 tile 网格（如 Windows Phone 7）虽然也是一览性的，但缺乏"布局作为自我表达"的维度
3. 中村强调的不是"信息展示效率"而是"构图美学"——这种从功能轴到内容轴再到美学轴的转变是他独有的

**中村自述**："普通智能手机有图标，不开应用就不知道内容。但想让人有某种程度上的'一览'。"（ITmedia, 2011）

**结果**：iida UI 成为 INFOBAR 最显著的差异化特征。Engadget 评价其"follows some similar glanceable design principles to Windows Phone 7"但强调了其可定制性和日本独有的审美。

**可提取规则**：
- **规则 4-A（一览性优先于层级）**：不要用图标隐藏内容。让用户"不点开就能看到"，降低信息获取的交互成本。新闻版面式的设计思维可以应用于各种信息展示场景。
- **规则 4-B（构图作为自我表达）**：当用户能自由布局模块时，布局行为本身就成为个性化的手段。工具的排列不仅是功能性选择，也是审美/身份表达。

### 决策 5：A01 -> A02 的范式深化：从"表达"到"物质质感"

- **来源**：一手（ITmedia A02 报道，中村发言）
- **可靠性**：高

**背景**：INFOBAR A01 的 iida UI 1.0 已经成功。但 A02 需要在已经很好的基础上更进一步。深泽直人提出了"羊羹"概念：硬件和软件不再是"红豆馅+最中皮"那样可以分离的二元结构，而是像羊羹那样——里外不分，质地均匀的一个整体。

**问题**：如何在 UI 2.0 中实现深泽的"羊羹"理念？1.0 已经实现了信息的"表出"（一览性）和用户的"表现"（布局定制），2.0 需要一个新的维度。

**决策**：中村为 iida UI 2.0 引入了一个全新的维度——**物质性的触感**。图标在滚动速度变化时会像ゼリー（果冻）一样变形——"ぷにぷに"（弹弹的）。这不是通过贴图实现的表面质感，而是通过算法生成的物理模拟行为。"不是表面的图形或纹理，而是通过算法的'物质的挙動'来表现这个产品独有的佇まい（存在姿态）。"（中村，ITmedia, 2013）

此外，中村还邀请音乐人 Cornelius（小山田圭吾）为 UI 设计了全部音效——"不只是 UI 动态的音效，还包括电话铃声、闹钟等，做了整体音效设计"。

**反决策分析**：为什么 2.0 不在信息密度、功能数量上竞争？
1. 功能竞赛是工程师思维。中村选择的是"质感深化"——1.0 解决了"看见什么"，2.0 解决"摸起来像什么"
2. 物理模拟的果冻变形不是装饰性的动画——它传达的是"这个产品有物质存在感"的理念，呼应深泽的"羊羹"概念
3. 邀请音乐家设计系统音效，把听觉维度也纳入"统一质感"的整体设计

**结果**：iida UI 2.0 的"果冻变形"成为 A02 最被讨论的交互特征。KDDI 田中社长发言："信息溢出的云时代，智能手机必须连'信息的面对方式'也一起设计。"

**可提取规则**：
- **规则 5-A（从表面到物质）**：当功能层面已经没有明显差异化空间时，向"物质质感"深挖——不是加更多功能，而是让已有功能"摸起来更好"。用算法模拟物理行为而非用贴图模拟表面。
- **规则 5-B（音效作为设计材料）**：UI 音效不是附加物——它是触觉反馈的听觉等价物。邀请音乐家以"作曲"的方式设计系统声音，把声音视为和图形同等重要的设计材料。

### 决策 6：硬件设计师和 UI 设计师的"双向同调"

- **来源**：二手（KDDI 砂原说明工作流程变化）
- **可靠性**：中高

**背景**：传统的手机开发流程是硬件设计师完成产品 -> 软件团队在硬件约束下开发 UI -> 两者几乎没有交集。

**问题**：如果追求"羊羹"概念（硬件软件不可分），传统流程中硬件设计师和 UI 设计师几乎没有交集，怎么可能做到同调？

**决策**：KDDI 在 INFOBAR A02 项目中打破常规，**安排了深泽直人（工业设计师）和中村勇吾（UI 设计师）多次直接讨论**。砂原（KDDI 产品企划责任人）描述："这是至今为止没有过的推进方式。"具体例子：机身左上电源键按下的瞬间，图标从其正下方"倏地"弹出；左侧音量键和功能键的高度与屏幕上对应 UI 元素的高度对齐——硬件按键和软件界面在空间上精确对应。

**反决策分析**：为什么不维持"硬件做完给软件"的传统流程？
1. 传统流程中硬件和软件是"馅和皮"的关系——你可以先做馅再做皮，或者反过来，但它们是两个独立物
2. "羊羹"要求硬件按键的位置/高度和屏幕 UI 元素的位置/高度精确匹配——这只有双方同时设计才能做到
3. 这种流程改变增加了组织和沟通成本，但换来了"所作（动作）的一贯性"——用户按压硬件按钮时，视觉反馈来自对应位置，形成肌肉记忆和视觉的闭环

**可提取规则**：
- **规则 6**：当追求硬件和软件的"一体化体验"时，必须打破组织壁垒——让工业设计师和 UI 设计师在同一时间讨论同一个交互场景。两者的设计不在上下游关系，而在平行同调关系。

## 三、ecotonoha — NEC 虚拟植树项目（2003）

### 决策 7：用"时间设计"替代"一次性事件"

- **来源**：一手（Shift 2004 采访，中村讲述 ecotonoha 设计意图）
- **可靠性**：高

**背景**：2003 年，NEC 想做 CSR（企业社会责任）宣传——用户贡献文字，公司承诺种真树。这是典型的"品牌互动 campaign"需求。

**问题**：大多数互动 campaign 是一次性的——访问、参与、离开。如何让用户不是"来过一次就忘"，而是持续回来？

**决策**：中村在设计中嵌入了精密的"时间设计"：
1. **每人每天只能写一片叶子**——"一天一枚"的节奏设计
2. **虚拟树的生长是累积的、可视的**——用户能看到自己的贡献和其他人的贡献如何一起改变树的形态
3. **"通过形状形成的社区导向"**——共同的视觉目标（一棵茂盛的树）驱动持续参与，而非通过文字/聊天

**中村自述**："Web 网站的节奏。设计时间。一个人每天只能出现一片叶子。作为日常习惯的 habitualization。这是在规律的时间间隔内进行的设计，这是电视等其他媒体无法实现的。"（Shift, 2004）

**反决策分析**：为什么不让用户一次写很多条消息？
1. 如果允许无限写，参与就从"仪式"变成"刷量"，失去意义感
2. "每天一片"的约束创造了一种稀缺性——用户会记住明天再来
3. 这是只有 web 能做到的事：持久的时间节奏。电视是一过性的，线下活动是间歇的。web 可以培养日常习惯。

**结果**：2003 年 7 月到 12 月运行期间，约 6 万条信息从世界各地发出。NEC 履行承诺在澳大利亚 Kangaroo Island 种植了 609 棵树。项目获得 2004 年戛纳 Cyber Lions Grand Prix（全场大奖）和 One Show Interactive Best of Show。

**可提取规则**：
- **规则 7-A（时间约束创造仪式感）**：限频（每人每天一次）不是功能限制，而是一种设计——它把参与从"一次性行为"转变为"日常仪式"。稀缺性 = 持续性。
- **规则 7-B（通过形状而非文字建立社区）**：用户围绕一个共同的、可见的、成长中的视觉目标（树）协作，这比围绕文字/话题的社区更能维持正面氛围和持续参与。

### 决策 8：一套界面解决全链路（概念表现到操作细节）

- **来源**：一手（Shift 2004 采访）
- **可靠性**：高

**背景**：ecotonoha 需要同时传达品牌环保理念、让用户理解参与机制、提供书写消息的界面、展示他人贡献。通常是多层页面/界面的组合。

**决策**：中村坚持 "To solve all the factors from the presentation of the entire concept to the details of 'How to use' with one interface. To keep it strictly 'simple'."（用一个界面解决全链路——从概念表达到操作细节。保持严格简单。）

树本身既是内容（概念表现），也是导航（点击树枝放大），也是输入界面（点击树叶写字），也是数据可视化（树的茂盛程度 = 参与度）。

**可提取规则**：
- **规则 8**：当概念和交互可以映射到同一个视觉隐喻时，不要把它们分离到不同界面。树 = 品牌理念 = 导航 = 数据 = 输入框。让用户在一个界面内完成"理解-操作-反馈"的闭环。

## 四、UNIQLO 系列项目（2006-2007）

### 决策 9：UNIQLO GRID —品牌互动 > 产品销售

- **来源**：一手（UNIQLO 官方项目说明）+ 二手（博客评论）
- **可靠性**：中高

**背景**：2007 年，UNIQLO 在伦敦牛津街开设第一家欧洲全球旗舰店。需要一个网站来提升品牌知名度，而非直接销售产品。

**问题**：大多数时尚品牌网站以产品展示和购买为核心。但 UNIQLO 此时的需求是"品牌认知"而非"转化率"。如何在网上制造品牌体验？

**决策**：中村设计了一个名为 UNIQLO GRID 的多人实时协作空间——用户在网格上分割、移动、旋转、组合 UNIQLO 的标志方块。多达 50 人同时在同一网格上互动，每个人可以编辑其他人的方块。核心交互逻辑：
- Logo 本身成为游戏材料（拆解/重组）
- 多人实时协作制造不可预测的集体创作
- 没有商业目标，纯粹是"品牌玩耍"

**UNIQLO 官方描述**："UNIQLO GRID 是一个让全世界用户一起参与、共同体验 UNIQLO 品牌的空间——通过分割、移动、旋转或组合 UNIQLO 标志。"

**反决策分析**：为什么不做电商或产品展示？
1. 时机：第一个欧洲旗舰店开业——此时需要的是让人"知道并记住 UNIQLO"，不是让人"在线购买"（欧洲电商基础设施尚不成熟）
2. Logo 解构/重组 = 品牌解构/重组 —— 用交互隐喻传达"UNIQLO 是你也可以参与塑造的品牌"
3. 多人实时协作在 2007 年是技术亮点，本身就能制造话题传播

**结果**：项目获得 One Show Silver 和 Tokyo Interactive Ad Awards Gold。

**可提取规则**：
- **规则 9**：当品牌目标是"认知"而非"转化"时，用交互游戏代替产品目录。让 logo / 品牌元素本身成为用户手中的玩具——拆解、重组、协作。品牌资产 > 产品资产。

### 决策 10：UNIQLO Explorer — 产品浏览 = 视觉模式重组

- **来源**：二手（netadreport 博客）+ 推断
- **可靠性**：中

**背景**：UNIQLO USA 网站需要一个产品浏览体验。但 UNIQLO 的产品（基础款服装）在视觉上不如时尚品牌引人注目——纯色 T 恤、基本款外套。

**问题**：如何让"浏览基础款服装"变得有趣？产品的视觉多样性不足，传统的 grid -> detail 模式会非常乏味。

**决策**：UNIQLO Explorer 采用了"交互式挂毯（tapestry）"概念——数百个产品面板排列成一个视觉织物。当用户点击查看某个产品后，**整个挂毯会根据刚刚浏览的产品的图案重新配置**。这意味着：
- 浏览行为改变了浏览环境本身
- 产品与产品之间建立了视觉上的"回声"关系
- 基础款的颜色/图案成为动态重组的数据源

**可提取规则**：
- **规则 10**：当产品本身的视觉多样性不足时，把浏览行为变成一种"模式生成"体验——用户点击不只是打开详情，而是改变了整个浏览环境的视觉构成。用系统行为代替产品本身的视觉丰富性。

## 五、NHK "Design Ah!" 教育节目（2011-至今）

### 决策 11：在保守公共广播约束下做激进创新

- **来源**：一手（6mirai 采访）+ 二手（Spoon & Tamago、Tokyo Art Beat）
- **可靠性**：高

**背景**：NHK 是日本公共广播机构，极其保守。儿童教育节目有严格的格式传统。"Design Ah!" 是一个关于"设计思维"的儿童教育节目，由平面设计师佐藤卓（总导演）、中村勇吾（影像导演）、Cornelius（音乐）三人组制作。

**问题**：公共广播的约束下如何做创新交互设计？NHK 的审查机制可能会否决"过于前卫"的视觉表达。

**决策**：中村和团队采用了"模块化短片段"策略：
- 每集 5-10 分钟的短片段格式
- 每个片段探索一个设计概念：对齐、线条、分解、观察、图案
- 部分片段**纯靠音乐和视觉传达抽象概念**——没有旁白，没有文字说明
- 利用公共广播的覆盖优势，把设计教育带入每个家庭

**关键互动**：中村在采访中特别提到总导演佐藤卓的工作方式——"佐藤先生的反应方式非常特别。他会非常兴奋地说'太好了！就这么做！'。一开始我还犹豫他是不是真心的，但渐渐地我发现自己被激活了，纯粹为了听到他说'太好了'而提出想法。"

**反决策分析**：为什么不用传统的"老师讲解+动画演示"教学格式？
1. 佐藤-中村-Cornelius 三人组本身就代表了打破分野——平面设计+交互设计+音乐同时在场
2. 纯视觉/音乐片段（没有语言）绕过了年龄和语言的障碍——学龄前儿童到成年人都能理解
3. 短格式降低了 NHK 的审查风险——如果某一段太激进，只影响 5 分钟而非整集

**结果**：Design Ah! 成为 NHK 最成功的设计教育节目之一，从电视扩展到展览（21_21 DESIGN SIGHT 的两个大型展览），影响了一代日本儿童的设计意识。截至 2024 年仍在播出。

**可提取规则**：
- **规则 11-A（模块化降低审批风险）**：在保守机构中，用模块化短片段替代长格式。每个模块独立可审查，激进片段和保守片段可以共存于同一产品中。
- **规则 11-B（正反馈创造者）**：领导者对创意的即时正面响应（"太好了！"）可以激活团队的创意产出。中村的经验：他后来发现自己纯粹为了听到这句话而提出更多想法。

---

### 决策 12：从电视到展览的媒介迁移——"能触摸的电视"

- **来源**：一手（中村在 6mirai 和 21_21 DESIGN SIGHT 的描述）
- **可靠性**：高

**背景**：Design Ah! 从电视节目扩展到 21_21 DESIGN SIGHT 的实体展览。电视是单向的、被动的，展览需要互动和身体参与。

**问题**：如何把"看电视"的体验转化为"动手体验"而不失去节目的趣味性本质？

**决策**：中村将展览定位为"游乐场和电视节目的融合"。关键设计决策：
1. **不分"艺术品"和"展品"**——不展示那些"看起来艺术"的东西，而是展示"可以触摸、可以感觉的有趣事物"
2. **"あ" in motion**：动作传感器记录孩子们跳舞的动作，实时在墙上生成动画——儿童的随机身体运动成为设计输入
3. **Dessin Ah!**：在 iPad 上画的东西会出现在主屏幕上，一笔一画被动画化——把儿童的涂鸦提升为展品
4. **沉浸室**：四面墙全投影的视听空间——身体被声音和影像包围

**中村自述**："我们在展览中不展示那些通常被认为值得在展览中展示的作品——那些明显具有艺术性的前卫作品。取而代之，我们专注于展示有趣的东西，可以被触摸和感受的东西。"（6mirai 采访）

**可提取规则**：
- **规则 12**：从屏幕迁移到物理空间时，不要试图"展示屏幕内容"。把屏幕的逻辑翻译为空间的身体互动——运动传感器、实时投影、触觉反馈。展览不是"观看设计"，而是"成为设计的一部分"。

## 六、早期实验作品与自我反思

### 决策 13：用个人实验室驱动商业创新

- **来源**：一手（多个采访：Shift 2001, Shift 2004, MoMA 2010, Design Indaba）
- **可靠性**：高

**背景**：中村在 1998-2004 年间维持了一个名为 "MONO*crafts" / yugop.com 的个人实验网站，同时为商业客户工作。这些实验包括物理模拟（弹跳球）、刚性体动力学、字体分解重构等。

**问题**：个人实验和客户工作之间的关系是什么？是"做完客户工作之后用剩余时间玩"还是"用实验喂养商业"？

**决策**：中村明确地将个人实验定位为"实验室"——探索 web 上每一种可能的视觉和交互表达方式。关键洞察：
- "有一段时间专注于客户工作，也有一段时间专注于实验和个人作品"
- yugop.com 上的物理模拟实验直接催生了 MoMA 网站的获选——策展人看到的就是那些"像物理世界一样有生命的球"
- MONO*crafts 2.0（1999）是"最早有效并令人信服地把'生命'——灵活性、可变性、可玩性、深度、速度——注入那时非常静态的 web 体验中的网站之一"（MoMA 数字媒体开发者的评价）

**中村自述**（关于 Flash 的立场）："我对 Flash 作为一个单纯的动画工具不感兴趣。我使用 Flash 是因为它的编程可能性、与图形图像集成的能力、良好的浏览器兼容性，以及易于使用。"（Design Indaba）

**可提取规则**：
- **规则 13-A（实验室-商业双周期）**：保持"实验季"和"客户季"的交替节奏。不要在客户工作中做实验（风险太大），也不要在实验季做客户工作（没有探索空间）。但实验季的产出必须达到可以展示的品质——策展人是用实验作品来选供应商的。
- **规则 13-B（工具态度）**：不忠于特定工具（Flash/HTML5/Unity），忠于工具能实现的表达可能性。中村使用 Flash 不是因为它是 Flash，而是因为它是"当时唯一能把图形+编程+浏览器兼容性整合在一起的工具"。

---

### 决策 14：承认并修正"过度复杂化"的倾向

- **来源**：一手（6mirai 采访自述，Humanity 开发自述）
- **可靠性**：高（自我批评）

**背景**：中村在多个项目中展现出把问题过于复杂化的倾向。

**中村自述**："有些时候我给自己制造困难。我似乎故意把事情搞复杂，然后后悔。或者我对员工提出太多要求，结果工作反而不好...我意识到当你和别人一起工作时，创造一个好的'流动'很重要。"

**具体案例**：

**案例 A（Design Ah! 展览）**：中村看到佐藤卓的协作方式后，反思自己的管理风格——"佐藤先生非常擅长创造好的流动"，而他自己有时因过多要求反而破坏产出质量。

**案例 B（Humanity 游戏开发）**："我们一直在挣扎。我提到过游戏是我们做的交互设计的延伸，但让游戏变得有趣和好玩的过程是非常独特的。即使有一些我认为可以忽略的意见，我们也勤奋地反复解决它们。我们会消除哪怕最轻微的压力，最终结果就是一个即使不能完全通关也令人愉快舒适的游戏。"中村承认游戏开发超出他最初的想象——"我相当早就意识到，创造有趣的游戏体验不是业余爱好者能轻松做到的。"

**反决策分析**：从这些自述中可以反向推导出中村认为"错的"做法：
1. 过度复杂化问题——"故意制造困难"然后"后悔"
2. 对团队过度要求——"太多要求"导致"产出反而不佳"
3. 过度自信导致低估陌生领域的难度——"我们过于乐观"，后来才请 Enhance 的资深游戏设计师介入

**可提取规则**：
- **规则 14-A（制造流动，而非制造困难）**：好的协作需要创造"好的流动"（good flow），而不是不断增加要求和复杂度。如果觉得事情越来越难，检查是否在自我制造不必要的难度。
- **规则 14-B（消除哪怕最小的压力）**：在打磨阶段，不要忽略任何微小的不适。团队不喜欢的细节，用户大概率也会感到不适。勤奋地消除每一个摩擦点。
- **规则 14-C（承认不擅长，尽早寻求专业帮助）**：当进入陌生领域时，不要用"设计思维万能"来掩盖技能差距。早点承认、早点请专家。

## 核心判断原则汇总

从以上 14 个决策中，可以提炼出贯穿中村设计方法论的核心判断原则：

### A. 关于"何时做减法/何时不修复"
- **放大模糊性**：当模糊性是内容的核心特质时，不要修复它——把它变成交互语言（决策 1）
- **匹配品质高度**：容器的野心必须匹配内容的野心（决策 2）
- **简化全链路**：用一个视觉隐喻承载概念+导航+操作+反馈（决策 8）

### B. 关于"如何传达动态设计意图"
- **代码=规格**：可运行原型的价值不在于展示，而在于消除解释空间（决策 3）
- **物理算法 > 表面贴图**：用算法模拟物质行为，而非用图形模拟外观（决策 5）

### C. 关于"信息架构的根本创新"
- **一览性优先**：不隐藏内容，降低信息获取成本（决策 4）
- **布局即表达**：当排列行为具有审美自由度时，它成为自我表达（决策 4）
- **产品浏览 = 模式生成**：当产品本身视觉单调时，让系统行为丰富体验（决策 10）

### D. 关于"时间与参与"
- **限频 = 仪式化**：约束不是限制功能，而是创造稀缺性和日常习惯（决策 7）
- **形状驱动的社区**：围绕共同视觉目标建立社区 > 围绕文字/话题的社区（决策 7）

### E. 关于"硬件与软件的关系"
- **双向同调**：打破组织壁垒，让硬件设计师和 UI 设计师同时工作（决策 6）
- **物质的举動**：通过算法赋予界面"物质存在感"——不只是看起来像，而是行为上符合物理直觉（决策 5）

### F. 关于"协作与自我修正"
- **制造流动**：好的领导 = 创造"好的流动"，不是不断增加要求（决策 14）
- **消除微小压力**：打磨阶段绝不放过任何一个摩擦点（决策 14）
- **承认无知**：在陌生领域不逞强，尽早请专业外援（决策 14）

### G. 关于"品牌 vs 电商"
- **认知阶段用游戏代替目录**：当目标是让用户"知道并记住"而非"现在购买"时，交互游戏 > 产品展示（决策 9）
- **Logo 解构 = 品牌解构**：让品牌标志成为用户的玩具——拆解、重组、协作（决策 9）

### H. 关于"机构约束下的创新"
- **模块化降低风险**：在保守机构中，用短格式模块替代长格式——每个模块独立可审查（决策 11）
