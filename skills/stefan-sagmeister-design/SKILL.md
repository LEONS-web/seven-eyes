# Stefan Sagmeister 情绪冲击方法论

**角色**：设计流水线特殊角色——不进日常流水线，仅在节日装扮/营销场景/强记忆需求时激活。

**使用前提**：仅在 product-context brief 明确要求强情绪记忆，且 THEME_PROMINENCE ≥ 8（强主题/节日装扮级）时调用。日常产品 UI 无需经过本层。

**最高门禁**：Sagmeister 不能绕过 product-context、hero asset 和基础可用性。情绪冲击只能加强产品主资产的记忆点，不能用节日符号、营销视觉或“纯美丽”元素遮挡、替代、降级主资产。对萌宠时光册而言，宠物照片仍必须是第一视觉落点。

**Sagmeister 管什么**：当 Scher 说"需要被记住"但还需要"被击中"的时候——Sagmeister 回答：怎么让用户心跳加速一秒。

**仲裁位置**：位于流水线侧翼，对节日装扮/营销场景输出情绪冲击判断，不参与日常流水线的线性仲裁。

## 决策启发式（4 条）

### H1 · 触摸心脏测试

**触发**：节日装扮或营销场景的设计方案，"专业"但"不够打动人心"。

**Sagmeister 原话**："Trying to touch the heart of the viewer. I've held that from the beginning."

```
1. 这个设计方案让我自己感觉到了什么？
   → 如果回答是"看起来不错"或"很专业" → 没通过。
   → 需要通过的情感是具体的：惊喜、温暖、怀旧、好奇、甚至轻微不适。

2. 它有没有一个"人"的痕迹？
   → Sagmeister："So much of graphic design pretends to be an answering machine."
   → 这个设计读起来像"系统自动生成的"还是"一个真实的人做给你的"？
   → 如果有手写、不完美、个人化的元素 → 加分。

3. 它有没有让观众"完成这个想法"的空间？
   → 不是把所有信息塞满。留一个缺口让观众自己跳进去。
   → Sagmeister 的"低功能设计"：不追求最优化，追求让观众参与完成。
```

**为什么**：Sagmeister 的核心信念——设计不只是解决问题（那是会计和工程师的事），设计可以让人愉悦、让人感动、让人记住。"Good design is design that either helps people or delights people."

**适用边界**：仅节日装扮/营销场景。日常工具型 UI 不需要"触摸心脏"——那是 Rams 的工具性格主场。

---

### H2 · 美丽即功能

**触发**：节日装扮方案"功能正确"但"不够美"。

**Sagmeister 原话**："Beauty is useless in the best way possible. It doesn't optimize for anything except the experience of being alive. Function gets all the credit in design, but beauty is what makes functionality matter to actual humans. It's what makes you remember something instead of just using it."

```
1. "这个方案的美丽在哪里？"
   → 如果回答是"配色协调""间距合理""层级清晰" → 那是 Refactoring UI 的落地质量，不是 Sagmeister 说的美丽。
   → Sagmeister 的美丽是有冲击力的、让你停下来看的、甚至有点"多余"的。

2. "删掉所有'多余'的美丽元素后，这个方案还剩下什么？"
   → 如果只剩下功能 → 它是一个好用的方案，但不是会被记住的方案。
   → 节日场景的目标不是"好用"，是"被记住"。

3. "它敢做'不必要'的事吗？"
   → 美丽本质上就是"不必要的"。一个按钮不需要渐变光晕——但加了之后用户会多停留 0.3 秒。
   → 节日装扮的预算：允许 20-30% 的视觉元素是"纯美丽、无功能"的。
```

**为什么**：Sagmeister 毕生论证的核心命题——美丽不是设计的装饰品，美丽是设计让人类在乎的原因。这在 Rams 的"尽可能少"和 Scher 的"被记住"之间提供了关键的补充：有些东西值得留下来，只因为它美。

**与 Rams 的冲突**：Rams 说"尽可能少的设计"——Sagmeister 说"有些东西值得留下来，只因为它美"。在节日场景，Sagmeister 赢。在产品日常 UI，Rams 赢。

---

### H3 · 诚实是最高级的设计策略

**触发**：节日方案感觉"刻意""用力过猛""假嗨"。

**Sagmeister 原话**："Everybody who is honest, is interesting." "I believe audiences can sense authenticity, and it fosters deeper connections and engagement with the work."

```
1. "这个方案在假装什么吗？"
   → 假装很欢乐？假装很温馨？假装用户会感动？
   → 如果团队自己看到方案不觉得开心/温馨/感动 → 用户在假装。

2. "有没有一个真实的、个人的东西可以放进去？"
   → Sagmeister 最著名的项目 "Things I've Learned in my Life So Far" 就是把个人日记变成公共艺术。
   → 节日方案里能不能有一个真实的用户故事？真实的数据？真实的照片？

3. "这个方案的'情绪'是从哪里来的？"
   → 套路化的节日符号（红灯笼、圣诞树、烟花 emoji）= "假"。
   → 独特的、个人的、甚至有点脆弱的表达 = "真"。
   → "假的节日气氛"比"没有节日气氛"更糟糕。
```

**为什么**：Sagmeister 的作品能"打动人"不是因为技巧——是因为他敢把自己放进去。当你把真实的自己放进设计时，观众能感觉到。假的设计——再漂亮——也只是"又一个节日皮肤"。

**适用边界**：诚实不等于没有节制的自曝。Sagmeister 的诚实是经过设计的诚实——选择性地暴露真实的脆弱来建立连接。不是"把所有个人问题都扔给用户"。

---

### H4 · 长期视角对抗短期噪音

**触发**：节日方案追逐"今年流行"的元素。

**Sagmeister 原话**："Short-term everything is crap, long-term everything is fantastic." "If we look at developments concerning the world from a long-term perspective, almost any aspect concerning humanity seems to get better."

```
1. "这个方案的视觉元素，明年还会成立吗？"
   → 如果依赖了"今年的 meme""今年的流行色""今年的热门 IP" → 有保质期。
   → Sagmeister 用几百年前的油画做当代数据可视化——跨越时间比追逐时间更有力。

2. "它是在对抗噪音，还是在增加噪音？"
   → 节日期间用户的注意力被大量营销内容轰炸。
   → 安静的美 > 大声的喊。一个真正美的方案不需要"喊"。

3. "它能不能成为一个'传统'？"
   → 最好的节日装扮不是一次性的——是用户每年期待回来看看的。
   → 如 Google Doodle 的长期演化——每个节日一个变体，但形成一个"期待的传统"。
```

**为什么**：Sagmeister 从 Steven Pinker 和 Danny Hillis（10000 年时钟）吸收了长期视角——在信息碎片化的时代，坚持做"放在一百年后仍然成立"的东西。"The latest tweets are just tiny blips in an overall rather healthy environment."

**与 Rams 的共振**：Rams 第 7 条"耐久"——不追逐时尚。Sagmeister 的长期思维与之高度一致，但出发角度不同：Rams 从"功能不应当被时尚绑票"出发，Sagmeister 从"人类长期在变好"出发。

```
→ 在本 skill 中跑 H1-H4
→ 输出情绪冲击判断给 huashu-design / 代码执行层
```

### 输入

| 字段 | 说明 |
|------|------|
| `product_context` | 产品目标、使用场景、强情绪表达是否被允许 |
| `hero_asset` | 必须保护的产品主资产 |
| `paula_scher_output` | 品牌记忆点方向 + 音量校准 |
| `theme_context` | 节日类型、目标情绪、文化语境 |
| `existing_assets` | 现有主题资产 |

### 输出（简化 5 字段）

1. **情绪基调一句话** — 用户看到的第一反应应该是什么？（惊喜/温暖/好奇/会心一笑）
2. **"触摸心脏"元素** — 方案中最能打动人的那个点是什么？如果没有 → 打回。
3. **美丽即功能清单** — 哪些元素是"纯美丽、无功能但必须保留"的？
4. **诚实性检查** — 有没有"假嗨"的嫌疑？如有，建议替换为真实的个人化元素。
5. **长期性判断** — 这个方案明年还能用吗？还是一个有保质期的"年度流行"？
6. **hero asset 风险** — 哪些情绪元素可能抢走主资产 C 位；必须降级、移位或删除的元素。

## 诚实边界

1. **只在节日/营销场景有效**。不适用于日常产品 UI、工具型界面、数据密集型页面。
2. **不替代 Scher 的品牌记忆点判断**。Sagmeister 在 Scher 基础上追加"情绪冲击力"——让记忆点不仅仅是"被记住"，而是"被击中"。
3. **"美丽"的判断高度主观**。Sagmeister 的方法论不能精确量化"美不美"——它提供一个方向（"敢做不必要的事"）而非一个公式。
4. **轻量提炼，不完整蒸馏**。本 skill 只有 4 条启发式，不包含完整的心智模型/输出契约/跨场景迁移体系。
5. **信息截止 2026-07-03**。
6. **不负责日常留存结构**。本层只能在强主题/节日/营销场景给情绪钩子，不能替代 Scher 的品牌记忆、Rams 的减法或 Refactoring UI 的落地战术。
