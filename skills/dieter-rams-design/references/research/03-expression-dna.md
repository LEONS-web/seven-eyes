## 1. 形式语言：Braun 产品的共同视觉基因

### 1.1 核心原则：秩序、和谐、经济（Order, Harmony, Economy）

R. Moss 在 1962 年对 Braun 设计的开创性分析（*Industrial Design* 杂志）中提炼出三条支配所有 Braun 设计的元规则：

- **秩序（Order）**："the methodic disposition of elements according to a definite system"——元素的系统化布置，不是"不邋遢"就能做到的，而是一个有意识地构建的系统。
- **和谐（Harmony）**："the special kind of order in which all parts are made to agree, creating a unity"——所有部分达成一致，创造统一性。这比秩序更难，因为它要求**体量与质量的平衡**（balance of masses and volumes made up of ordered elements）。
- **经济（Economy）**：不浪费任何视觉元素。

> **🔍 UI 判断规则**：一个界面是否构成"系统"而非元素的随意堆叠？检查：所有组件是否共享同一种秩序逻辑（间距系统、对齐规则、层级比例），还是"各说各话"？

### 1.2 Braun 2018 年设计语言（官方 10 项原则）

Braun 设计总监 Oliver Grabes 领导团队在 2009 年将 Rams 的 DNA 编码为一套可操作的设计语言，至今仍在 Kronberg 设计工作室使用：

1. **Pure & Fluid Geometry（纯粹与流动的几何）**：独立产品基于清晰的建筑化形态（architectural shapes）；手持产品通过连接基本几何形与平滑过渡创造流动几何。
2. **Symmetry & Direction（对称与方向性）**：产品捕捉人类的对称美感，同时通过方向性指示来"解释产品的使用方式"。
3. **Order & Balance（秩序与平衡）**：设计元素排列在几何网格内，具有清晰的功能层级，产生视觉清晰度和和谐外观。
4. **Reduction（减省）**：视觉元素减至最少，强调产品的可用性和功能性。
5. **Interface Elements（界面元素）**：所有交互元素——按钮、开关——被设计为**圆形或拉长的圆形（racetrack/circle）**，只有这两种形态；其比例、颜色和材质随功能变化。
6. **Geometric Parting Lines（几何分模线）**：分模线界定产品的操作区域，创造视觉结构，特征是几何化的、受控的对齐方式。
7. **Distinctive Shapes（独特轮廓）**：基于建筑或人体工学形态，每个产品有辨识度的轮廓。
8. **Iconic Details（标志性细节）**：独特细节赋予产品不可替代的市场存在感，同时将其识别为 Braun 产品。
9. **Colors & Materials（色彩与材质）**：主色为黑、白、灰和金属色。色彩强调仅用于突出产品功能或操作控件。
10. **Clear Product Graphics（清晰的产品图形）**：功能性、极简的图形语言，跨所有产品类别一致。

> **🔍 UI 判断规则**：这 10 条是直接从物理产品翻译到 UI 的最强桥梁。每条都可以变成检查问题，例如："这个界面的所有交互控件是否只有两种形态（主/次，如同 circle/racetrack）？" "分模线（卡片边界、section 分割）是否完全几何对齐？"

### 1.3 SK-4 "白雪公主之棺"（1956）——Braun DNA 的零号样本

SK-4（Phonosuper SK4）由 Hans Gugelot 和 Dieter Rams 联合设计，Wilhelm Wagenfeld 参与唱机底盘开发。它是 Braun 设计语言的原点。以下是它建立的 DNA 要素：

#### 结构创新
- **U 形金属外壳 + 两块木侧板**：Gugelot 的构造创意——"两块垂直木侧板，一块金属板张拉其间"——成为他贯穿所有设计的 *idée fixe*（固定观念），本质上是"承重与非承重功能的视觉戏剧化"。
- **金属外壳取代木壳**：这是历史上第一次有人敢用金属做收音机外壳。工程师抱怨，但认识到其必要性。结果是：电子产品不再需要伪装成家具。
- **透明亚克力盖**：Rams 最初加这个盖是因为原金属盖在测试中会振动——**功能驱动的材料选择**。盖子的透明性让内部机械结构可视化，这一决定后来定义了整个唱机行业的形态。"白雪公主之棺"（Schneewittchensarg）的绰号来自白色外壳 + 透明盖的组合。

#### 操作面板布局
- 控制面板位于顶部右侧——功能与美学思考共同决定的位置，类似 1930 年代 Braun Super 6740W 的布局。
- 按键和旋钮的排列**不是从外部配件商购买的现成件**，而是根据设计师的想法专门开发和制造的。

#### 材料组合
- 涂装金属（painted metal）+ 木材（最初是枫木/白榉木争议，最终用白蜡木）+ 塑料（透明亚克力）。
- 尺寸：24.1 × 58.4 × 29.2 cm ——一个紧凑的矩形比例。

> **🔍 UI 判断规则**：SK-4 的启示是——(1) 让功能结构可见，不要隐藏；(2) 任何装饰性决定的背后应该是功能需求；(3) 产品不需要伪装成它不所是的东西（电子产品不需要伪装成家具）。

### 1.4 "安静的几何"（Quiet Geometry）的具体含义

Rams 在 1976 年 Vitsœ 演讲中的原话：

> "When we concentrate on the essential elements in design, when we omit all superfluous elements, we find forms become: **quiet, comfortable, understandable and, most importantly, long lasting.**"

这不是一个可量化的形式参数，而是一种**设计态度**：

- **不尖叫**：产品不争夺注意力。Vitsœ 的家具"不喊叫；它以相对匿名的方式执行功能"。
- **不支配**："Furniture should not dominate, it should be quiet, pleasant, understandable and durable."
- **为用户的自我表达留空间**：产品是中性的、克制的，像一个空容器，让用户的生活内容成为主角。

Braun 2018 年设计语言补充了"安静的几何"的操作化表述：
- 独立产品 = 清晰的建筑化形状（硬边直角）
- 手持产品 = 流体几何（基本几何形 + 平滑过渡）

> **🔍 UI 判断规则**：一个 UI 是否"安静"？不看它的静态截图，看它是否"争夺注意力"：有没有不必要的动画、过多的颜色变化、过大的视觉比重元素？界面应该像 SK-4 的外壳——安静地退到背景，让内容（用户的数据、任务）成为焦点。

### 1.5 圆角半径逻辑

Braun 产品并非没有圆角，但圆角的逻辑是：
- **大产品的硬边**：桌面级产品（收音机、音响）采用几乎直角的边缘，强调建筑感。
- **手持产品的流体几何**：剃须刀、吹风机、美容仪器使用平滑过渡的曲面，因为人体工学要求如此。
- **界面元素只有两种形态**：正圆形（circle）或拉长的圆形/跑道形（racetrack/elongated circle）。没有第三种。

> **🔍 UI 判断规则**：UI 中的按钮/卡片圆角是否遵循一致的系统？建议：(1) 大容器（卡片、面板）= 极小圆角或零圆角；(2) 交互控件（按钮）= 统一的圆角半径（如全圆 pill 或统一小圆角）；(3) 禁止多于两种圆角半径同时存在于一个界面。

## 2. 色彩与材质体系

### 2.1 核心调色板（附 hex 值）

来源：Hue Atlas 从 Braun 产品摄影、Vitsœ 606 档案和 Braun 1960s-1990s 工业色彩标准中提取。以下是最接近 Rams 原始意图的数字近似值：

**主色系（Achromatic Ground）：**

| 名称 | Hex | 用途 | 说明 |
|------|-----|------|------|
| Braun White | `#F0EDE5` | 主要外壳、顶面 | 暖调灰白，带有微弱黄灰底色，在日光下读作中性但避免纯白的临床冰冷感 |
| Braun Grey | `#8A8A87` | 次级表面、旋钮、格栅 | 接近中性的中灰，带微弱暖底调，用于需要功能性"后退"的表面 |
| Pebble | `#C5C3BE` | 浅色变体、按钮表面 | 介于 Braun White 和 Braun Grey 之间，分辨交互表面与外壳 |
| Anthracite | `#2E2E2C` | 深色变体、底座 | 极暗暖灰，充作"近黑"但没有纯黑的刺目感 |
| Warm Black | `#1A1A18` | 字体、logo、标签 | 近黑带微弱暖底，仅用于排版元素 |

**强调色（Functional Accents —— 每次只出现一个）：**

| 名称 | Hex | 用途 | 说明 |
|------|-----|------|------|
| Function Yellow | `#D4B018` | 电源指示灯、单控高亮 | 哑光金色黄，在低光下保持可见性 |
| Signal Red | `#C02820` | 电源按钮、警告指示 | 轻度去饱和的红，避免纯红对产品宁静感的冲击 |

**金属表面（非色值）：**
- 自然或深色阳极氧化铝（natural or dark anodised aluminium）
- 绒面铬（velour chrome）——极少使用，仅用于天线或提手

### 2.2 色彩的功能逻辑

Rams 对色彩的态度不是"极简主义作为审美偏好"，而是"极简主义作为信息设计"：

1. **彩色 = 操作信号**：自 1960 年代以来，几乎所有 Rams 产品至少有一个亮色元素。这些有色特征**始终**在按钮开关上具有"清晰可辨的指示功能"。
2. **功能色彩编码系统**：Braun 音频设备的色彩图表定义了七个功能色分配——FM 按钮 = "血橙"（blood orange），唱机按钮 = "蜜黄"（honey yellow），所有电源按钮 = "叶绿"（leaf green）。这些颜色与调谐刻度盘上的色彩对应，用户不需要阅读说明书即可理解。
3. **产品越大，色彩越克制**：大型、技术复杂的产品（如 audio 308 紧凑系统）使用更克制的亮色。小型手持设备（剃须刀、吹风机、闹钟）才出现更大胆的色彩。
4. **色彩限制的刻意性**：RT 20 收音机只提供两种色彩方案（白+山毛榉木，炭灰+梨木），"不会推出五种或十种"。Vitsœ 椅子系列的亚麻面料只有 4 种柔和色调，皮革只有 6 种颜色。**例外或定制不被允许。**
5. **强调色的隔离规则**：Red 和 Yellow 从不一起出现在同一产品上。在同一屏幕/产品上使用两种强调色会"使每个颜色的功能信号崩溃"。

> **🔍 UI 判断规则（直接可操作）**：
> - 主背景必须来自无彩色系：优先 `#F0EDE5`（亮主题）或 `#2E2E2C`（暗主题）
> - 所有文本 = 一种颜色：`#1A1A18`（亮底）或其反相
> - 交互表面（输入框、卡片）使用 `#C5C3BE` 通过色调偏移传递可供性信号
> - 每屏最多 1 个强调色元素（CTA 或状态指示器），Red `#C02820` 或 Yellow `#D4B018`，绝不同时出现
> - "如果一个颜色没有功能理由，它就是视觉噪音"

### 2.3 哑光 vs 亮面的选择逻辑

- **哑光优先**：Rams 明确拒绝"高光泽、冷白色表面"——它们反射环境光、模糊形态，与"设计应该澄清而非复杂化"的原则冲突。
- **亮面仅用于功能理由**：亮铬仅出现在天线和提手上，且"非常克制和审慎地"使用。
- **黑色转向（1965）**：Studio 1000 hi-fi 系统标志着一个转折——用炭灰色结构漆（anthracite-coloured structured lacquer）涂覆除正面外的所有面，配以拉长圆角铝条而非螺丝。黑色使音响单元看起来更密集紧凑，体现技术强度。此后黑色主导消费电子设计 30 年。

> **🔍 UI 判断规则**：UI 中避免纯黑（`#000000`）和纯白（`#FFFFFF`）——两者都是"高反射"的数字等价物。用 `#1A1A18` 和 `#F0EDE5` 替代。阴影/投影的使用应"不引人注目"——如铝条取代螺丝的逻辑。

### 2.4 材料选择逻辑：塑料/金属/木材

- **金属**：外壳的主体材料（从 SK-4 开始）。涂装金属板、阳极氧化铝、绒面铬。传达"诚实"——材料看起来就是它实际所是。
- **木材**：最初用于 SK-4 侧板（枫木→白蜡木）。Rams 的祖父是木匠，他自己做过木工学徒。木材更多出现在早期作品和扬声器中（木质外壳音质更好）。后期减少使用。
- **塑料**：透明亚克力（SK-4 盖）、热塑性塑料（桌面打火机主体）、凸面按键（ET 系列计算器）。塑料用于需要触感、透明性或复杂成型的部位。
- **核心原则**：材料选择由功能驱动，不为装饰。Rams 说过："我对那些把技术设备伪装成客厅家具的做法没有好感。"

> **🔍 UI 判断规则**："材料诚实"的数字翻译：(1) 不要用拟物化装饰（假皮革纹理、假金属渐变）；(2) 如果某个 UI 元素看起来可点击，它必须真的可点击；(3) 不要用阴影和渐变来伪装一个平面 div 为"按钮"——用清晰的形态（圆/跑道形）和色调偏移来定义交互区。

## 3. 比例与模数

### 3.1 Vitsoe 606 通用搁架系统的模数逻辑

606 系统（1960 年设计，至今仍在生产）是 Rams 模数思维的终极体现：

**核心模数单元：**
- **E-Track（铝制 E 型轨道）**：系统的唯一结构组件，挤出成型铝材，直接固定在墙上。
- **两种架宽**：窄 = 66.7cm（可用宽度 65.5cm），宽 = 91.2cm（可用宽度 90cm）。
- **标准 E-Track 高度**：57cm、114cm、171cm、200cm、228cm（大致呈倍数关系）。
- **连接系统**：带凹槽的铝销（notched aluminium pins）滑入 E-Track 位置——无需工具。

**模数不是数学比例，是"功能模数"：**
窄宽与宽宽的比例约为 1:1.37（接近 √2 = 1.414，即 ISO 纸张比例），但 Rams 从未宣称使用黄金比例或任何数学定式。他的模数来自**功能约束**：架子的宽度必须容纳特定物品（窄架放书籍，宽架放 Braun Audio 1 hi-fi 系统）。606 系统最初就是为容纳 Braun Audio 1 的尺寸而设计的，使音响和书架可以"作为一个考虑好的整体坐在一起"。

**适应性与长寿：**
- 1970 年代购买的组件与今天制造的完全兼容（E-Track 销系统未变）。
- 一面 5 米宽的墙有 27 种可能的架宽组合。
- 50+ 种不同的组件可整合进系统。
- 架子可以三种方向悬挂：正面朝上、倒置、或竖转 90° 成为磁性"公告板"。

> **🔍 UI 判断规则**：606 的模数哲学是——(1) 定义最小结构单元（E-Track = 你的 12 列栅格或 8pt 间距系统）；(2) 所有组件必须插入同一结构语言；(3) 不追求数学完美，追求"功能约束决定比例"；(4) 向后兼容性——旧内容必须能在新布局中正常工作。

### 3.2 比例系统的实际操作

Rams 在 1976 年演讲中阐述了比例的核心：
> "Design at Vitsœ brings all individual elements into proportion. An often-cited feature of the Vitsœ collection is its balance, its harmony, its belonging together."

关键洞察：
1. **没有黄金比例的刻意追求**：Rams 从未像 Le Corbusier 的 Modulor 那样宣称使用 ϕ（黄金比例）。他的比例感来自建筑训练和木工实践。
2. **"与人的平衡比例"（balanced proportion with people）**：Rams 将比例定义为产品与人体尺寸、感官和理解力的和谐。这比几何比例更深刻——是人体工学比例。
3. **产品之间的比例一致性**：1963 年的 TS45、TG60 和 L450 每个单元尺寸完全相同，用户可以自由组合（垂直或水平堆叠）。这不是美学比例——是系统比例。
4. **模块化尺寸设计**：单元被设计为可以垂直或水平堆叠的模数尺寸。

> **🔍 UI 判断规则**：(1) 间距和尺寸来自功能约束（内容需要多少空间），而非装饰性比例；(2) 所有"模块"（卡片、section、面板）应可互换排列；(3) 比例一致性 > 比例完美性——如果一个产品系列中的两个界面放在一起，它们应该"看起来属于同一个系统"。

### 3.3 "好设计是克制的"在比例上的体现

"Weniger, aber besser"（少，但更好）在比例上的操作化：

- **体量压缩**：1965 年 Studio 1000 系统用黑色让音响"看起来更密集紧凑"——比例感不仅来自尺寸，也来自色彩分配。
- **没有多余的空间**：SK-4 的操作面板与唱机之间的空间分配由功能决定，没有"留白装饰"。
- **克制的组件数量**：606 系统用最少的结构组件实现最大的灵活性——E-Track + 销 + 架板 = 整个系统。

> **🔍 UI 判断规则**：检查每个界面：是否存在"装饰性留白"（没有信息功能的空间）？间距是否用来"让东西看起来更好"而非"让东西更清晰"？如果移除一个间距元素功能不受影响，它就该被移除。

## 4. 字体与信息层级

### 4.1 字体选择：Akzidenz-Grotesk

**历史路径：**
- Otl Aicher 在 1955 年发起使用 Akzidenz-Grotesk（由 HfG Ulm 为 Braun 做视觉识别时）。
- Wolfgang Schmittel（Braun 传播总监，1952-1981）在 Rams 监督下创建了几乎 100 页的详细品牌手册。
- Akzidenz-Grotesk（1898 年首发）是 Helvetica（1957）和 Univers 的前身，属于 grotesque / neo-grotesque 无衬线字体家族。
- SK-4 的早期标签实际上使用了 Nobel 字体，从 SK-5 起统一为 Akzidenz-Grotesk。

**为什么是 Akzidenz-Grotesk？**
- 无衬线 = 无装饰 = 诚实。在当时的德国，竞争者还在用情感化的图像和修饰性字体。
- 可读性强，在任何尺寸下都清晰可辨。
- 系统化使用：大写/小写、重量变化建立层级，而非使用不同字体。
- 2022 年 Braun 委托 Iconwerk 设计了新企业字体 **Braun Linear**——"线宽有最小调制"，带有大圆角与直线融合。

> **🔍 UI 判断规则**：UI 字体的选择逻辑——(1) 只用一种无衬线字体族；(2) 层级通过字号、字重、间距建立，不用第二种字体；(3) 不用斜体（Akzidenz-Grotesk 长期没有意大利体，层级靠字重而非倾斜）。

### 4.2 "什么信息必须被看到 / 什么可以隐藏"

Rams 的产品有一个严格的信息可见性逻辑：

**必须被看到的：**
- 功能标识（FM、Phono、On/Off、音量）——以最小文字 + 颜色编码呈现
- 调谐刻度——大面积清晰图形（T1000 的刻度以"极大且清晰"著称）
- 品牌标识——低调但存在

**可以被隐藏的：**
- 技术规格（放在盒内或背面）
- 操作说明书（产品本身就是说明书——操作逻辑"不言自明"）
- 紧固件（螺丝被拉长的圆角铝条取代）
- 背部接线（SK-4 的金属板包裹前、顶、后三面，组件从底部插入，使得设备可以放在房间中央作为家具）

**Braun 视觉手册的具体信息层级规则（来源：onlyonceshop.com 对原手册的分析）：**
- 字号以点数指定并转换为毫米
- 行距以数字定义，基线到基线的距离与字母形态本身同等重要
- 层级通过**缩放和分离**创建，而非样式化：标题更大但"不更吵"；重要信息通过留白隔离，而非装饰
- 正文保持左对齐，让节奏而非对称引导阅读
- 价格卡系统完全模数化：定义好的文字尺寸、固定层级、可预测的间距。红色被"克制且一致地"使用，几乎像技术信号而非品牌花饰。价格在前，产品名称随后，描述安静地放在下面。

> **🔍 UI 判断规则**：
> - 信息必须有"可见性层级"：第一眼看到什么？第二眼？第三眼？测试方法——眯眼看界面，什么还能被识别？
> - 任何可以放在"二级界面"（设置、更多选项、背面）的信息都不应该出现在主界面上
> - 留白不是装饰，是信息隔离工具——两组信息之间的间距大小应反映它们的逻辑关系强度

### 4.3 "好设计是易懂的"（Understandable / Selbst-erklärend）

这是 Rams 第四原则的操作化：

**物理产品如何实现"不用说明书就能操作"：**
1. **颜色功能编码**：FM = 橙色按钮 + 橙色刻度标记 —— 视觉链接消除理解负担
2. **凸面按键**：ET 系列计算器的凸面按键产生高光——触觉 + 视觉双重信号
3. **操作键形态统一**：所有交互元素只有两种形态（圆/跑道形），用户不需要学习识别新形态
4. **功能分组**：ET 44 计算器（1978，与 Dietrich Lubs 合作）将按键按功能组色彩编码——算术运算一种颜色，记忆功能另一种颜色。眼睛在阅读文字标签之前就能区分数学类别。"这不是装饰；这是通过色彩进行沟通。"
5. **物理约束**：只有相关控件可见。T1000 世界接收器的前面板金属盖可以拆卸，内部印有操作信息——用户在操作时才获得信息。

> **🔍 UI 判断规则**：测试一个 UI 是否"易懂"——(1) 新用户能否在不阅读任何文字标签的情况下完成主要任务？(2) 视觉分组（颜色、间距、形态）是否在文字之前就传达了关系？(3) "说明书"内容是否可以在用户需要时才展示（tooltip、渐进披露）而非一直占据屏幕？

## 5. 从物理产品到数字 UI 的翻译

### 5.1 可直接对应的原则

以下 Rams 十诫可以几乎一字不改地用于数字产品评审：

| 原则 | 物理产品中的体现 | 数字 UI 中的直接对应 | 判断问题 |
|------|------------------|---------------------|----------|
| **#2 有用** | 产品服务于明确目的 | App/网站有一个清晰的主要任务 | 离开这个功能，用户还能完成核心任务吗？ |
| **#4 易懂** | 操作不言自明 | 界面可自学（onboarding 不是必需的） | 首次用户能不借助帮助完成任务吗？ |
| **#5 不突兀** | 产品是工具，不是装饰品 | UI 不争夺注意力 | 界面元素中是否有"炫耀性"的动画/装饰？ |
| **#6 诚实** | 产品不做它不能兑现的承诺 | UI 不夸大功能、不伪造紧迫感 | 有没有 dark pattern？有没有假稀缺性？ |
| **#8 彻底到最后一个细节** | 螺丝、铰链、所有元件都经过设计 | 所有状态（空、加载、错误、边缘案例）都被设计 | 加载状态的 spinner 是浏览器默认的吗？ |
| **#10 尽可能少的设计** | 减少到本质，不为非必要元素所累 | 每个元素服务于用户目标 | 删除这个元素，功能是否会受损？ |

### 5.2 需要"翻译层"的原则

| 原则 | 物理产品的体现 | 翻译难度 | 数字 UI 翻译 |
|------|--------------|---------|-------------|
| **#1 创新** | 新技术催生新形态（透明亚克力盖因振动问题而生） | 中 | 技术创新应为用户问题服务，不为创新而创新。数字 UI 中的"创新"常常表现为不必要的交互模式变更。 |
| **#3 美学** | 产品日常使用影响人的幸福感 | 中 | UI 美学不仅是"好看"——是能否在每天 8 小时使用中不让人视觉疲劳。检查：调色板的 "视觉持久性" 。 |
| **#7 持久** | 材料耐久 + 向后兼容 + 不过时 | 高 | 数字产品"持久"是什么？代码可持续维护、设计系统不随潮流摇摆、不追逐每年 redesign。一个 2015 年的组件在今天是否仍能正常工作？ |
| **#9 环保** | 材料可回收、避免浪费、反对有计划废止 | 最高 | 数字产品"环保"的翻译：代码效率（减少能耗）、数据节制（不收集不必要的数据）、设备兼容性（不让用户必须购买新设备）、认知可持续性（不制造信息过载）。 |

### 5.3 在数字产品中不适用或需要重新定义的原则

- **"模数化尺寸以便物理堆叠"** → 翻译为"响应式系统中的组件可组合性"，但物理堆叠的不可能性意味着这部分 DNA 需要全新诠释。
- **"表面处理逻辑（阳极氧化铝 vs 涂装金属 vs 绒面铬）"** → 数字 UI 中的"表面处理"等价物是：阴影、边框、背景色、透明度。核心原则保持不变——"一种材料对应一种功能信号"→ 一个视觉处理对应一个交互状态。
- **"分模线"（Parting Lines）** → 在数字 UI 中最接近的是：卡片边界、分割线、section 分隔。Braun 的规则——分模线必须"几何化、受控对齐"——直接意味着：所有分割必须对齐到栅格，没有随意的 1px 偏移。

### 5.4 核心翻译原则总结

从 Rams 物理设计到数字 UI 的翻译，核心不是模仿"看起来像 Braun"，而是继承**决策逻辑**：

| Rams 的决策逻辑 | 数字 UI 等价 |
|----------------|-------------|
| "这个材料为什么在这里？" | "这个颜色/阴影/边框为什么在这里？" |
| "去掉这个元素，产品还能用吗？" | "去掉这个元素，用户还能完成任务吗？" |
| "用户不看说明书能操作吗？" | "用户不读 onboarding 能使用吗？" |
| "这个亮色在承担什么通信功能？" | "这个强调色在传达什么状态/优先级？" |
| "五年后这个产品还好看吗？" | "这个 UI 在下一个设计趋势到来时还需要翻新吗？" |
| "所有组件来自同一个系统吗？" | "这个界面用的是同一个设计系统的组件吗？" |
