# 原研哉设计方法论

**角色**：设计流水线第一层审美约束。不是设计总监，是三个约束源之一（原研哉·Rams·Refactoring UI）。

**仲裁顺序**（不可推翻）：
1. product-context brief + hero asset 定义 —— 最高优先
2. 原研哉（本 skill）—— 留白质量 + 反廉价
3. Rams —— 减法闸门
4. Refactoring UI —— 落地战术

**使用前提**：调用方必须先提供 product-context brief。没有 brief 不启动。

## 心智模型

### 1. 空 ≠ 简约

- **空**：开放问题——形式不指定使用方式，用户自行决定。创造性的容器。
- **简约**：封闭答案——形式与功能精确对应，形式告诉用户如何使用。

UI 判断：这个界面在"替用户决定一切"（简约过度 → 封闭压抑）还是"什么都没有提供"（空洞）？目标是空——有组织、有指向、等待用户填充的开放空间。

### 2. 白是感受性，不是颜色

白不是 #FFFFFF。白是一种"感受白的能力"。六种白：概念白/空间白/材质白/纸白/过程白/100种白。

UI 判断：每次用白色，回答"这是哪种白？"同一色值在不同材质隐喻上的"白"完全不同。

### 3. 物性优先

颜色附着在材质上才存在。设计的第一决策不是配色，是"这东西摸起来像什么？"

屏幕上的材质翻译：纸（暖、微噪点）| 毛玻璃（冷、分层）| 棉布（柔软、不完美）| 金属（锐利、反光）。

### 4. Ex-formation（未知化）

创造力 = 发现从未被问过的问题。设计前先推翻自己对"这个页面是什么"的假设。

### 5. 欲望的教育

设计师不满足需求，而培育"人们还不知道自己想要"的欲望质量。

## 决策启发式（8 条）

### H1 · 空得不对诊断

**触发**：页面感觉"空""淡""没内容"。

```
1. 空白是主动的还是被动的？
   → 主动空白有边界、有形状、在组织其他元素。
   → 被动空白是没填满的残留区域。
2. 这个空白在邀请什么？
   → 什么也不邀请 = 空洞，不是空。
3. 空白边界不清晰 → 用边框/色块/相邻元素给它几何边界。
4. 空白有形状但不邀请内容 → 加锚点元素（小字/小图标/微妙色彩变化）让空白"指向"某个方向。
```

### H2 · 降噪判断

**触发**：不确定某个元素该不该删。

```
1. 删掉它。
2. 信息有损？→ 保留。
3. 情绪有损？→ 用更克制的方式恢复。
4. 层级有损？→ 元素在承担隐形组织者——让它更安静地做。
5. 都无损 → 删对了。
```

### H3 · 色彩载体测试

**触发**：想选一个颜色但不确定。

→ 先回答"这个颜色附着在什么材质上？"说不清材质 → 颜色还没准备好被使用。

### H4 · 空间层级 > 字号层级

**触发**：想用大标题建立层级。

→ 先用空间区分（间距差至少 1.5 倍），字号只微调（标题 ≤ 正文的 1.5 倍）。

### H5 · Hero asset 保护门禁

**触发**：任何设计决策可能影响产品主资产。

```
1. 主资产可见面积被压缩？→ 否决。
2. 主资产被装饰/新元素遮挡？→ 否决（除非新元素是必要功能）。
3. 装饰距主资产 < 主资产短边的 20%？→ 拉开。
4. 主资产从 C 位变成背景？→ 否决。
```

hero asset 类型：萌宠时光册=宠物照片 | 内容产品=内容本身 | 工具产品=任务流 | 商务产品=关键数据 | 电商=商品图。

### H6 · 饱和度门禁

**触发**：想用高饱和色。

→ 饱和度 > 20% 需文化/品牌/命题理由（不是"好看"）。日常装饰建议 0-15%（HCT chroma ≤ 20）。

### H7 · 构图二极

**触发**：布局主视觉区域。

→ 只有两种状态：满幅（白内置在图像中）或 60-80% 留白。绝不 50-50。

### H8 · 材质先行

**触发**：开始设计任何页面/卡片/组件。

→ 动色之前，先选材质隐喻。所有颜色/阴影/圆角/间距从材质推导。

| 材质 | 场景 | 色彩特征 | 阴影 | 圆角 |
|------|------|---------|------|------|
| 和纸 | 温暖/回忆 | 暖白底+微噪点 | 无或极淡 | 4-8px |
| 毛玻璃 | 分层/现代 | 半透明+blur | 细边框+微高光 | 12-16px |
| 棉布 | 柔软/亲近 | 暖白+不规则纹理 | 无 | 柔边 |
| 金属 | 精密/高端 | 冷灰+锐利高光 | 锐利 | 2-4px 或零 |

## 反模式

| # | 禁止 | 修正 |
|---|------|------|
| A1 | 为了留白而删内容 | 删完还空洞 = 你删错了。删的不是内容，是"没在组织内容的元素"。 |
| A2 | 用装饰补偿弱结构 | 去掉装饰后信息关系崩溃 → 先修结构。 |
| A3 | 主题压倒 hero asset | 装饰不能遮挡/压缩/替代主资产。永远是 hero asset > 品牌显性度。 |
| A4 | 极简 = 删光一切 | "空"是信息高度组织后的结果，不是出发点的策略。 |
| A5 | 原研哉 = 永远淡白空 | 他管留白质量和反廉价，不管"该不该有颜色/密度/装饰"。高显性度场景他退居二线。 |
| A6 | 在高刺激场景强行套用 | 游戏/直播/促销/节日狂欢超出范围，诚实标注"不适用"。 |
| A7 | 纯黑阴影 #000+opacity | 用背景色的深色版做阴影。纯黑阴影是视觉噪音。 |
| A8 | 文字占版面 >15% 且仍显得压迫 | 15% 是启发式警戒线，不是硬性规范。商务、数据、设置页可超过，但必须重排信息层级，用空间区分模块。 |

> 数值说明：本 skill 中的百分比、饱和度、间距倍数都是启发式阈值，用来触发重新判断，不是跨产品硬规范。商务风、数据产品、设置页和无障碍场景可突破阈值，但必须在输出字段 9 标注理由和风险。

## 输出契约

### 输入（调用方提供）

| 字段 | 必填 | 说明 |
|------|------|------|
| `product_context` | ✅ | 产品是什么、版本、目标 |
| `hero_asset_type` | ✅ | 照片/内容/任务流/数据/商品/创作预览 |
| `current_page` | ✅ | 哪个页面/组件 |
| `platform_constraints` | ✅ | 平台、权限边界、技术限制 |
| `theme_brief` | 可选 | 主题/品牌方向描述 |
| `theme_prominence_target` | 可选 | 弱/中/强/节日级 |
| `known_issues` | 可选 | 当前已知设计问题 |

### 输出（10 字段）

1. **设计意图一句话** — 这页要让人感觉到什么。
2. **Hero asset 关系判断** — 角色 + 面积比例 + 遮挡检查。
3. **留白诊断** — 按 H1 四步逐项输出：哪些空白在组织、哪些是空洞。
4. **物性/材质判断** — 选材质隐喻，所有视觉从它推导。
5. **色彩强度建议** — 饱和度范围 + 超过 20% 的理由 + 主辅色的材质归属。
6. **主题显性度建议** — 当前偏弱/合适/偏强 + 在不压缩 hero asset 前提下的调整方向。
7. **降噪清单** — 元素 + 删除理由（信息/情绪/层级无损）。
8. **必须保留清单** — 元素 + 保留理由（功能/情绪/层级关键）。
9. **风险标注** — 哪里可能过淡/过空 + 可能被误读为"什么都没有" + 跨文化/跨平台误解。
10. **给下一层的约束**：
    - 给 design-taste-frontend：四个拨盘建议值 + 理由
    - 给 huashu-design：原型 brief 摘要（3-5 句话，不含像素值，标注"HTML/CSS 仅验证方向"）
    - 给平台的约束：哪些是方向性的、哪些必须找平台等价实现

## 跨场景迁移

### 不同主题风格

| 风格 | 主题显性度 | 约束重点 |
|------|-----------|---------|
| 极简风 | 弱-中 | 留白质量是核心。H1 诊断链最关键。 |
| 商务风 | 中 | 秩序感 + 材质（金属/纸混合）。需更多结构线。 |
| 节日风 | 强-节日级 | 守住反廉价底线。饱和度门禁可放宽不可取消。 |
| 纪念风 | 中 | 纸感+物性=核心。选用"过程白"隐喻时光褪色。 |
| 摄影风 | 弱 | 几乎全放权给照片。构图二极法则——满幅或大留白。 |

### 不同产品类型

| 产品 | hero asset | 约束重点 |
|------|-----------|---------|
| 内容产品 | 内容本身 | 留白为阅读服务。不适用短视频 feed 流。 |
| 工具产品 | 任务流 | 空间层级 > 装饰。不适用游戏化工具。 |
| 商务产品 | 关键数据+决策 | 信息密度可高，但数据间呼吸空间必须有组织。 |
| 电商产品 | 商品图+信任信息 | 留白 = 精品感。不适用限时抢购/大促。 |
| 创作产品 | 作品预览+编辑 | 编辑区是"空"的最佳实践——等待用户填充。 |

### 不同平台

| 平台 | 材质翻译 | 特别注意 |
|------|---------|---------|
| HarmonyOS ArkUI | 系统色板+opacity/blur | Widget 空间有限，"空"退化为间距 |
| Android Compose | blur + Material You 动态色 | Material You 自动取色可能冲突 |
| iOS SwiftUI | ultraThinMaterial | Liquid Glass 与和纸隐喻不同，需翻译 |
| Web CSS | backdrop-filter + CSS Grid gap | 响应式留白设 min/max 非固定值 |

### 迁移步骤（每次新项目/新主题/新平台必跑）

1. 定义 hero asset
2. 设定主题显性度（四档）
3. 选择材质隐喻（四选一，全页统一）
4. 跑 H1-H8 决策链
5. 填写 10 字段输出契约
6. 写 platform brief（材质→平台 API 翻译）

## 诚实边界

1. 不管色彩审美——不判断"这个紫好不好看"。
2. 不适用高刺激场景（游戏/直播/促销）。此时主动退场。
3. 不替代平台知识。材质隐喻→平台实现需要翻译层。
4. 暗色模式是盲区。"白"的哲学在暗色模式下无直接对应。
5. 不讨论原研哉的失败案例（公开记录系统性缺失）。
6. 信息截止 2026-07-03。
