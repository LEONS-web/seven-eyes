# 设计师方法论 Skill 通用审查清单

> 创建时间：2026-07-03  
> 用途：审查 `projects/design-pipeline/skills-src/*/SKILL.md` 是否能进入设计流水线。  
> 适用对象：Paula Scher、Susan Kare、Yugo Nakamura、Dieter Rams、Refactoring UI 等后续蒸馏产物。  

## 0. 总结论分级

- `通过`：可进入下一层流水线或 huashu-design 原型阶段。
- `小修后通过`：方向正确，但需要补 1-3 个关键边界、迁移说明或输出字段。
- `需返工`：有可用材料，但还不是可执行 skill，无法稳定指导设计。
- `不通过`：人格模仿、传记摘要、语录合集，或会伤害 product-context / hero asset。

审查时先写风险，再写优点。

## 1. 方法论而非人格模仿

合格：

- 用心智模型、启发式、反模式、输入输出契约组织。
- 不写“我作为某某设计师会怎样”。
- 传记、作品、访谈只作为方法来源，不作为正文主体。
- 能说明该人物负责流水线里的哪一层判断。

不合格：

- 大段生平介绍。
- 堆名言和代表作品。
- 只提风格特征，不给可执行判断。
- 让 AI 模仿人物语气做设计。

## 2. 必须服从 product-context 和 hero asset

每个 skill 必须明确：

- product-context brief 是最高优先级。
- hero asset 不能被该人物方法论压缩、遮挡、替代或降为背景。
- 人物方法论只能调节视觉策略，不能改变产品核心目标。

通用 hero asset 示例：

- 宠物相册：宠物照片。
- 内容产品：内容本身。
- 工具产品：任务流、输入结果和关键操作。
- 商务产品：关键数据、信任信息和决策动作。
- 电商产品：商品、价格、信任和购买路径。
- 创作产品：作品预览和编辑控制。

## 3. 必须有清楚的流水线位置

每个 skill 应声明自己负责什么、不负责什么。

当前推荐分工：

- 原研哉：留白质量、物性、反廉价、空白是否有组织。
- Paula Scher：品牌记忆点、视觉冲击、主题显性度。
- Susan Kare：图标语言、微情绪、亲和力、低分辨率识别。
- Yugo Nakamura：交互节奏、动效质感、系统性 motion grammar。
- Dieter Rams：功能减法、诚实设计、长期可用性。
- Refactoring UI：间距、字重、阴影、边框、色彩等落地战术。

如果一个 skill 试图包办所有层级，判定为 `需返工`。

## 4. 必须能跨主题、跨产品迁移

合格 skill 不能只会修当前小女巫主题。至少要说明：

- 用在极简风时，哪些参数会降低。
- 用在商务风时，如何避免变成无聊的模板。
- 用在节日风时，如何提高显性度但不破坏核心任务。
- 用在其他 App 时，如何先替换 hero asset，再复用判断框架。
- 用在 HarmonyOS / Android / iOS / Web 时，哪些结论需要平台翻译。

## 5. 必须能处理小女巫验证案例，但不能被它绑定

当前 Phase 验证案例仍然是萌宠时光册小女巫首页。

审查时至少问：

- 它能否解释 App 内页面为什么“空得不对”？
- 它能否避免主题过淡、缺乏留存记忆点？
- 它是否保护宠物照片 C 位？
- 它是否能帮助 huashu-design 出 A/B/C 变体？

但正文不能写成“小女巫专用经验包”。

## 6. 输出契约必须稳定

每个 skill 至少要有：

- 输入字段：product context、hero asset、页面/组件、目标、上游输出、约束边界。
- 输出字段：该层判断、风险、必须保留、必须降低、给下一层的约束。
- 反模式：常见误用和纠偏方式。
- 迁移指南：主题、产品、平台三类迁移。
- 诚实边界：这个方法论不擅长什么。

如果没有输出契约，只能作为参考文章，不能作为流水线 skill。

## 7. 数值只能是启发式

允许写音量、密度、时长、比例等数值，但必须说明：

- 数值是决策启发式，不是跨产品硬规范。
- 平台约束、任务效率和可读性可以覆盖数值。
- 最终以 product-context、hero asset 和视觉验收为准。

## 8. 不把公共标准塞进人物 skill

以下内容应沉淀到 `standards/`，不应重复塞进每个人物 skill：

- 反 AI slop 通用清单。
- product-context brief 模板。
- UX / 可访问性底线。
- 资产映射协议。
- HTML 原型到平台实现 brief。
- 视觉评审 checklist。

人物 skill 可以引用这些标准，但不应复制一整套公共规则。

## 9. 可进入下一步的最低标准

一个新设计师 skill 至少满足以下条件，才建议进入 Phase 4 质量验证：

- 不是人格模仿。
- 有 3-6 个清晰心智模型。
- 有 6-10 条可执行启发式。
- 有 6-10 条反模式。
- 有输入输出契约。
- 有跨主题、跨产品、跨平台迁移说明。
- 有诚实边界。
- 明确服从 product-context 和 hero asset。
