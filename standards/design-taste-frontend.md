## 一、四个拨盘（核心配置）

每次设计前，基于 product-context brief + 第一层设计师约束输出，设定四个拨盘值。

### 1.1 DESIGN_VARIANCE（设计偏差度）

**定义**：1 = 完全对称/标准布局，10 = 艺术化/非对称/实验性。

**通用含义**（不绑定任何主题）：
| 值 | 含义 | 适用场景 |
|----|------|---------|
| 3-4 | 高度标准化、可预测 | 公共部门、无障碍关键场景、企业后台 |
| 5-6 | 现代干净但不出格 | 开发者工具、SaaS、内容阅读 |
| 7-8 | 有作者意图的差异化 | 品牌产品、消费者 App、设计师作品集 |
| 9-10 | 实验性、艺术驱动 | 创意机构、艺术展览、实验项目 |

**小女巫推荐值**：**7**
理由：萌宠时光册是消费者产品，不能像企业后台一样标准。但也不到实验艺术——需要品牌记忆点但不吓人。

**迁移规则**：
- 极简风：5-6（偏差来自材质质感，不是布局实验）
- 商务风：4-5（可预测性 > 差异化）
- 节日风：8-9（强主题可以更艺术化）
- 工具产品：3-5（效率优先，不搞视觉实验）

---

### 1.2 MOTION_INTENSITY（动效强度）

**定义**：1 = 静态/零动效，10 = 电影级连续动效。

**通用含义**：
| 值 | 含义 | 适用场景 |
|----|------|---------|
| 1-2 | 零动效或仅系统默认 | 无障碍优先、公共服务 |
| 3-4 | 仅按压反馈 + 一次性页面入场 | 企业后台、数据密集产品 |
| 5-6 | 按压 + 入场 + 选定状态切换 + 滚动触发 | 主流消费者 App |
| 7-8 | 连续氛围动效 + 手势跟随 + 多层视差 | 品牌产品、高端消费者 |
| 9-10 | 全电影级、物理引擎驱动 | 游戏、节日活动页、概念展示 |

**小女巫推荐值**：**4**
理由：一次性柔光或按压反馈可以，但持续飘落（当前问题）在真机上易缺失、易廉价、易干扰。动效在这里不是主力——照片和主题是主力。动效是"给质感加一层"，不是"替代表演"。

**迁移规则**：
- 极简风：2-3（动效少但质感高——一个微妙的 300ms 淡入比五个花哨动效更"极简"）
- 商务风：1-2（无动效或仅系统默认转场）
- 节日风：6-7（节日氛围可以要更多动效，但必须受 Yugo Nakamura 方法论约束）
- 游戏化产品：7-9

---

### 1.3 VISUAL_DENSITY（视觉密度）

**定义**：1 = 极稀/美术馆式留白，10 = 极高密度/仪表盘。

**通用含义**：
| 值 | 含义 | 适用场景 |
|----|------|---------|
| 1-2 | 每屏 1-3 个元素，大量留白 | 奢侈品牌、艺术展示、品牌宣言 |
| 3-4 | 每屏 3-5 个信息块，留白充足 | 消费者产品首页、品牌落地页 |
| 5-6 | 每屏 5-8 个信息块，留白有组织 | SaaS 首页、内容 feed、设置页 |
| 7-8 | 每屏 8-15 个信息块，呼吸空间紧凑 | 数据分析、仪表盘、专业工具 |
| 9-10 | 每屏 15+ 个信息块 | 金融交易终端、航空管制 |

**小女巫推荐值**：**5**
理由：首屏需要照片舞台 + 主题相框 + 标题 + 操作入口 + 底部 Tab。不是极简（不是 3），也不是数据仪表盘（不是 8）。中密度，用相框、纸感、照片舞台组织首屏，避免大块无组织空白。

**迁移规则**：
- 极简风：2-3（密度低但每个元素精准）
- 商务风：6-7（中高密度但秩序强）
- 节日风：4-5（主题元素多但信息层级不能乱）
- 数据产品：7-9

---

### 1.4 THEME_PROMINENCE（主题显性度）

**定义**：主题/品牌视觉在多强地表达自己。这是流水线新增的第四拨盘（原 design-taste-frontend skill 只有前三个）。

**通用含义**：
| 级别 | 值 | 含义 | 适用场景 |
|------|-----|------|---------|
| **弱主题** | 1-3 | 主题元素极少，产品 hero asset 占绝对 C 位。主题只在微妙线索中存在（底色、字体、间距） | 极简风、摄影风、工具产品 |
| **中主题** | 4-6 | 主题元素可感知但不抢戏。用户知道"有主题"但不会因此分心 | 日常风、商务风、内容产品 |
| **强主题** | 7-8 | 主题是主要视觉特征，形成留存记忆点。hero asset 仍然 C 位但主题舞台很强 | 小女巫目标、纪念风、品牌产品 |
| **节日装扮级** | 9-10 | 临时节日换装，强视觉冲击。hero asset 可以暂时让步但不可被替代 | 节日活动、限时换装、营销页 |

**小女巫推荐值**：**7（强主题）**
理由：小女巫是 1.5.0 的留存钩子。主题必须成为记忆点——用户打开不是因为"该喂宠物了"，而是"想看看宠物在小女巫花园里"。但不能到 9-10（节日级），因为这不是限时活动，是日常主题。照片仍然 C 位。

**迁移规则**：
- 极简风：1-3（主题显性度最小化，靠材质和间距表达差异）
- 商务风：3-5（中主题，品牌色+秩序感，不抢数据）
- 节日风：8-10（可到节日装扮级，但必须加回来检查 hero asset 是否被替代）
- 新 App：必须由 product-context brief 决定值，不可沿用小女巫默认值

## 二、仲裁顺序与拨盘联动

### 2.1 优先级

```
product-context brief 中的 THEME_PROMINENCE 目标（最高）
  ↓
原研哉 skill 输出的"给 design-taste-frontend 的约束"
  ↓
本层四个拨盘设定
  ↓
反 slop 规则检查（拨盘不能把设计推到禁区）
```

### 2.2 拨盘联动规则

- THEME_PROMINENCE ≥ 7 → VISUAL_DENSITY 至少 4（强主题需要足够的舞台空间）
- THEME_PROMINENCE ≤ 3 → DESIGN_VARIANCE ≤ 6（弱主题时布局不要太实验，否则没有锚点）
- MOTION_INTENSITY ≥ 6 → 必须有 Yugo Nakamura 方法论约束（当前阶段：需要显式标注"动效风险"）
- VISUAL_DENSITY ≥ 7 → 必须过原研哉 H1 诊断（高密度下的留白是否仍然有组织）

## 三、反 AI Slop 规则

### 3.1 萌宠时光册专用反 slop

| # | 禁止 | 为什么 | 正确做法 |
|----|------|--------|---------|
| S1 | 紫色/蓝色渐变科技感 | 萌宠是陪伴+温度，不是 SaaS/AI 产品 | 暖色系：淡藤紫、奶白、暖灰、蜡笔色 |
| S2 | emoji 或假魔法贴纸感 | 贴纸 = 廉价 = 用户感知"这是AI画的" | 用 Susan Kare 式符号语言：小图标有像素级亲和力 |
| S3 | 每个标题配装饰 icon | 多余装饰稀释真正重要的信息 | 标题靠空间层级区分，不靠 icon 标记 |
| S4 | 为"主题感"牺牲照片识别 | 宠物照片是唯一 hero asset，主题不能替代它 | 主题是舞台，照片是演员。舞台再好看也不能挡住演员 |
| S5 | 浅紫白卡片无限堆叠 | "有紫但没有设计"——颜色≠品牌识别 | 每张卡片要有材质隐喻（纸/毛玻璃/棉布），颜色从材质推导 |
| S6 | 系统级持续飘落动效 | 真机上易缺失、易廉价、易干扰用户操作 | 一次性柔光入场 + 按压反馈。如需要氛围动效，用 3-5 秒自动结束的微动效 |
| S7 | 纯黑阴影（#000 + opacity） | 不携带品牌信息，在留白丰富的页面上是视觉噪音 | 用背景色/卡片色的深色版做阴影（如淡紫卡片 → 深紫灰阴影） |
| S8 | 圆角混乱（有的方有的圆有的全圆角） | 材质隐喻不一致 → 视觉噪音 | 选定材质后统一圆角：纸=4-8px / 毛玻璃=12-16px / 棉布=柔边 / 金属=2-4px 或零 |

### 3.2 通用反 slop（跨主题/跨产品/跨平台）

| # | 禁止 | 为什么 |
|----|------|--------|
| G1 | 紫色渐变科技感作为任何新项目的默认 | AI 训练语料里"SaaS/AI 落地页"的万能公式，= 品牌识别度归零 |
| G2 | 把原研哉误用为"所有产品都淡/白/空" | 原研哉只管留白质量和反廉价，不管"该不该有颜色"。高显性度场景他退居二线 |
| G3 | 把 Refactoring UI 误用为通用 SaaS 卡片模板 | Refactoring UI 是战术工具包，不是视觉风格 |
| G4 | 把 Dieter Rams 误用为"所有产品只能有 3 个按钮" | Rams 是减法闸门——判断该不该减，不是命令你减到只剩骨头 |
| G5 | 为"设计感"削弱产品 hero asset | 好看≠好用。hero asset 永远第一 |
| G6 | 不读平台约束就出视觉稿 | HTML/CSS 原型只能验证方向，不能假装是 ArkTS/Compose/SwiftUI 像素稿 |
| G7 | Inter + slate-900 作为默认字体组合 | 这是 LLM 默认值，不携带品牌信息 |
| G8 | 每个新项目默认居中 Hero + 三列等宽卡片 | 这是另一个 LLM 默认值，不等于"设计了" |

## 四、ArkTS 落地约束

本层输出给下游的设计方向必须遵守以下约束，不可要求：

- ❌ 新增权限（READ_IMAGEVIDEO 等）
- ❌ 相册扫描 / 静默读取
- ❌ 网络素材 / CDN 图片 / 在线字体
- ❌ 账号系统 / IAP / 云同步
- ❌ 替换 `formImages(fd) + memory://widgetPhoto` 主链路

✅ 可以要求的（ArkUI 本地可做）：
- Stack 分层：照片底层 + 相框中层 + 装饰顶层
- 色块、渐变、边框、borderRadius 圆角
- 静态图片资源（本地 PNG/JPEG，`$r()` 引用）
- 透明度 + blur 叠加（ArkUI 的 `opacity` + `backdropBlur`）
- 一次性状态动效（入场淡入、按压 scale 反馈、选中高亮过渡）
- 列/行间距、padding、margin（通过 `space` + `padding` + `constraintSize`）
- 自定义字体（本地 .ttf 文件，不超 500KB）

⚠️ 有条件可尝试（需验证真机性能）：
- `Image` 的 `objectFit` + `renderMode` 控制照片显示
- `transition` + `animateTo` 做一次性入场
- `linearGradient` 做材质光效模拟

## 五、与第一层（原研哉 skill）的对接

本层接收原研哉 skill 输出字段 10 中的 `给 design-taste-frontend 的约束`，具体包括：

- 材质隐喻建议（纸/毛玻璃/棉布/金属）→ 影响 DESIGN_VARIANCE 和 VISUAL_DENSITY
- 色彩强度建议（饱和度范围）→ 影响反 slop 规则的具体阈值
- 留白诊断结论 → 影响 VISUAL_DENSITY 的上下调整
- 风险标注 → 转化为本层的"特别注意"清单

本层不直接引用原研哉的心智模型或决策启发式——那是第一层的事。本层只管把第一层的约束翻译成四个拨盘值 + 反 slop 清单。

## 六、与第三层（原子 skill）的对接

本层输出给第三层的固定格式：

| 输出字段 | 说明 | 示例 |
|---------|------|------|
| `DESIGN_VARIANCE` | 拨盘值 + 一句话理由 | 7: 消费者产品需要品牌记忆点但不吓人 |
| `MOTION_INTENSITY` | 拨盘值 + 动效策略方向 | 4: 一次性柔光入场+按压反馈，禁止持续飘落 |
| `VISUAL_DENSITY` | 拨盘值 + 密度组织策略 | 5: 中密度，用相框/纸感/照片舞台组织 |
| `THEME_PROMINENCE` | 拨盘值 + 主题策略 | 7: 强主题，主题是留存钩子但照片仍C位 |
| `MATERIAL_METAPHOR` | 材质隐喻 | 和纸+毛玻璃混合：主卡用和纸（暖/回忆），底部操作区用毛玻璃（轻/分层） |
| `ANTI_SLOP_CHECKLIST` | 必须过的反 slop 清单 | S1-S8 + G1-G8 中适用于当前页面的项 |
| `PLATFORM_CONSTRAINTS` | 平台硬约束 | §四 全部适用 |
| `CRITIQUE_SENSITIVE_POINTS` | critique 必查项 | 照片面积是否被压缩、留白是否有组织、动效是否在真机上测试 |

## 七、小女巫首页首屏 · 完整参数卡

> 这是 Phase 1 验证案例的完整参数快照。**这些值是小女巫专用的，不可直接复制到其他主题。** 其他主题必须按 §一 的通用定义重新设定。

| 参数 | 值 | 一句话理由 |
|------|-----|-----------|
| DESIGN_VARIANCE | 7 | 品牌感需要差异化布局，但不过度实验 |
| MOTION_INTENSITY | 4 | 一次性柔光入场+按压反馈，不要持续飘落 |
| VISUAL_DENSITY | 5 | 中密度，用相框和纸感组织，消除无组织空白 |
| THEME_PROMINENCE | 7 | 强主题，小女巫是留存钩子，但照片仍 C 位 |
| MATERIAL_METAPHOR | 和纸主卡 + 毛玻璃底部区 | 回忆温度（纸）+ 现代分层（毛玻璃） |
| HERO_ASSET | 宠物方形照片 | 不可被遮挡/压缩/替代 |
| ANIMATION_STRATEGY | 入场：300ms 淡入。按压：scale 0.97。选中：背景色过渡 200ms | 无持续动效 |
| COLOR_PALETTE | 主：暖白底（#FDF9F5 附近）+ 淡藤紫（#E8DCF0 附近）。辅：蜡笔绿点缀 | 饱和度控制在 15% 以内 |
